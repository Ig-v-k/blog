    How to Set Up a Private Maven Repository in Amazon S3

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Amazon S3 - отличное место для хранения частных артефактов Maven. Предположим, что вы храните общедоступные артефакты в Maven Central, потому что хотите, чтобы они были доступны всем. Частные артефакты - это те, которые вы не хотите видеть никому, кроме членов вашей команды. Таким образом, вы хотите разместить свои файлы .jar там и убедиться, что они видны только вашей команде. Вот как мы делаем это во всех наших Java-проектах на нашем сайте.

    Create an S3 Bucket

    Сначала создайте новое хранилище S3. Я рекомендую вам назвать его с использованием домена вашего проекта и префикса. Например, с repo.teamed.io, repo является префиксом, а teamed.io - доменом.

    Нет необходимости настраивать какие-либо разрешения для этого хранилища. Просто создайте его через консоль Amazon S3.

    Create an IAM User

    Создайте нового пользователя IAM. Я рекомендую вам назвать его teamed-maven, если имя вашего проекта teamed.

    Добавьте новую “встроенную политику” пользователю:

    {
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:*",
      "Resource": [
        "arn:aws:s3:::repo.teamed.io",
        "arn:aws:s3:::repo.teamed.io/*"
      ]
    }
  ]
}

    Здесь repo.teamed.io - это название S3-ведра, которое вы создали минуту назад.

    Убедитесь, что у вас есть “ключ доступа” для этого нового пользователя. Он должен выглядеть примерно так:

    key: AKIAI9NNNJD5D7X4TUVA
secret: t5tZQCwuaRhmlOXfbGE5aTBMFw34iFyxfCEr32av

    Ключ состоит из 20 символов (все заглавные буквы), а секрет состоит из 40 символов.

    Extend settings.xml

    Добавьте эту конфигурацию в ваш файл ~/.m2/settings.xml:

    <settings>
  <servers>
    <server>
      <id>repo.teamed.io</id>
      <username>AKIAI9NNNJD5D7X4TUVA</username>
      <password>t5tZQCwuaRhmlOXfbGE5aTBMFw34iFyxfCEr32av</password>
    </server>
    [...]
  </servers>
  [...]
</settings>

    Configure pom.xml

    Добавьте эту конфигурацию в pom.xml:

    <project>
  <distributionManagement>
    <snapshotRepository>
      <id>repo.teamed.io</id>
      <url>s3://repo.teamed.io/snapshot</url>
    </snapshotRepository>
    <repository>
      <id>repo.teamed.io</id>
      <url>s3://repo.teamed.io/release</url>
    </repository>
  </distributionManagement>
  <repositories>
    <repository>
      <id>repo.teamed.io</id>
      <url>s3://repo.teamed.io/release</url>
    </repository>
  </repositories>
  [...]
</project>

    Затем, настройте S3 Wagon, также в pom.xml:

    <project>
  <build>
    <extensions>
      <extension>
        <groupId>org.kuali.maven.wagons</groupId>
        <artifactId>maven-s3-wagon</artifactId>
        <version>1.2.1</version>
      </extension>
    </extensions>
    [...]
  </build>
</project>

    Вы готовы к работе. Вы можете развернуть свои артефакты, просто запустив Maven из командной строки.

    $ mvn clean deploy

    Configure s3auth.com

    Теперь вы хотите просмотреть эти артефакты в браузере в защищенном режиме, предоставив безопасные учетные данные. Я рекомендую использовать s3auth.com, как объясняется в статье Basic HTTP Auth for S3 Buckets.

    Configure Rultor

    Еще одна рекомендация состоит в настройке rultor.com для автоматического развертывания ваших артефактов в S3.

    Сначала зашифруйте свой settings.xml с помощью этого удаленного инструмента Rultor:

    $ gem install rultor
$ rultor encrypt -p me/test settings.xml

    Вместо me/test вы должны использовать имя вашего проекта на GitHub.

    В результате вы получите новый файл с именем settings.xml.asc. Добавьте его в корневую директорию вашего проекта, затем сделайте коммит и отправьте изменения. Файл содержит вашу секретную информацию, но только сервер Rultor может расшифровать ее.

    Создайте файл .rultor.yml в корневой директории вашего проекта (формат этого файла подробно описан на странице справки Rultor).

    decrypt:
  settings.xml: "repo/settings.xml.asc"
deploy:
  script: |
    mvn clean deploy --settings ../settings.xml

    Теперь пришло время увидеть, как все это работает вместе. Создайте новый тикет в трекере проблем GitHub и отправьте что-то вроде этого в него (подробнее о командах Rultor можно прочитать здесь):

    @rultor deploy

    Вы получите ответ в течение нескольких секунд. Остальное будет сделано Rultor’ом.

    That’s it.
