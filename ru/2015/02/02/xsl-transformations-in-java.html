    XSL Transformation in Java: An Easy Way

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    XSL преобразование (XSLT) - это мощный механизм для преобразования одного XML документа в другой. Однако в Java манипуляции с XML довольно длинны и сложны. Даже для простого XSL преобразования вам придется написать несколько десятков строк кода - и даже больше, если требуется правильная обработка и журналирование исключений. jcabi-xml - это небольшая библиотека с открытым исходным кодом, которая значительно облегчает жизнь, позволяя выполнять разбор XML и навигацию по XPath с помощью нескольких простых методов. Давайте посмотрим, как эта библиотека помогает в XSL преобразованиях.

    Во-первых, рассмотрим практический пример - [rultor.com] (https://www.rultor.com) - это хостинговый помощник DevOps, который автоматизирует операции по выпуску, слиянию и развертыванию. Rultor сохраняет каждую сессию разговора с конечным пользователем (также известную как “talk”) в записи DynamoDB. В каждом разговоре есть несколько ситуаций, которые нужно обрабатывать; поэтому использование нескольких столбцов записи не является практичным. Вместо этого мы храним только несколько параметров каждого разговора в столбцах записи (например, идентификатор и имя) и помещаем все остальное в один столбец XML.

    Приблизительно так выглядит наша таблица DynamoDB:

    +----+---------------+--------------------------------------+
| id | name          | xml                                  |
+----+---------------+--------------------------------------+
| 12 | jcabi-xml#54  | <?xml version='1.0'?>                |
|    |               | <talk public="true">                 |
|    |               |   <request id="e5f4b3">...</request> |
|    |               | </talk>                              |
+----+---------------+--------------------------------------+
| 13 | jcabi-email#2 | <?xml version='1.0'?>                |
|    |               | <talk public="true">                 |
|    |               |   <daemon id="f787fe">...</daemon>   |
|    |               | </talk>                              |
+----+---------------+--------------------------------------+

    Когда пользователь размещает @rultor status в GitHub-заявке, Rultor должен ответить полным отчетом о текущем разговоре. Чтобы создать такой текстовый ответ (обычный пользователь не оценит XML-ответ), нам нужно извлечь столбец xml из соответствующей записи DynamoDB и преобразовать его в обычный английский текст.

    Вот как мы делаем это с помощью jcabi-xml и его класса XSLDocument.

    final String xml = // comes from DynamoDB
final XSL xsl = new XSLDocument(
  this.getClass().getResourceAsStream("status.xsl")
);
final String text = xsl.applyTo(xml);

    Вот и всё. Теперь давайте посмотрим, что находится в файле status.xsl (это всего лишь его каркас; полная версия находится здесь).

    <xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
  version="2.0">
  <xsl:output method="text"/>
  <xsl:template match="/talk">
    <xsl:text>Hi, here is your status report:</xsl:text>
    ...
  </xsl:template>
  <xsl:template match="node()|@*">
    <xsl:copy>
      <xsl:apply-templates select="node()|@*"/>
    </xsl:copy>
  </xsl:template>
</xsl:stylesheet>

    Это хорошая практика создавать документы XSL только один раз во время работы приложения. У нас есть статический метод утилиты XSLDocument.make() для этого:

    final class Foo {
  private static final XSL STYLESHEET = XSLDocument.make(
    Foo.class.getResourceAsStream("stylesheet.xsl")
  );
  public XML style(final XML xml) {
    return Foo.STYLESHEET.transform(xml);
  }
}

    Обратите внимание на то, что мы используем XSLT 2.0. Встроенная реализация XSLT на Java не поддерживает версию 2.0, и для ее работы мы используем две зависимости Maven Saxon.

    <dependency>
  <groupId>net.sourceforge.saxon</groupId>
  <artifactId>saxon</artifactId>
  <version>9.1.0.8</version>
  <scope>runtime</scope>
</dependency>
<dependency>
  <groupId>net.sourceforge.saxon</groupId>
  <artifactId>saxon</artifactId>
  <version>9.1.0.8</version>
  <classifier>xpath</classifier>
  <scope>runtime</scope>
</dependency>

    Все, что вам нужно сделать, чтобы начать использовать jcabi-xml для XSL-трансформаций, это добавить эту зависимость в ваш pom.xml:

    <dependency>
  <groupId>com.jcabi</groupId>
  <artifactId>jcabi-xml</artifactId>
</dependency>
