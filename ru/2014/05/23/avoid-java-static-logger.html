    Get Rid of Java Static Loggers

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Это очень распространенная практика в Java (использование LoggerFactory из slf4j):

    import org.slf4j.LoggerFactory;
public class Foo {
  private static final Logger LOGGER =
    LoggerFactory.getLogger(Foo.class);
  public void save(String file) {
    // save the file
    if (Foo.LOGGER.isInfoEnabled()) {
      Foo.LOGGER.info("file {} saved successfuly", file);
    }
  }
}

    Что не так с этим? Дублирование кода.

    Это статическое свойство LOGGER должно быть объявлено в каждом классе, где требуется вести запись журнала. Всего несколько строк кода, но на мой взгляд, это просто шум.

    Чтобы облегчить жизнь, я создал библиотеку примерно два года назад, jcabi-log, в которой есть удобный класс утилит Logger (да, я знаю, что классы утилиты - это зло).

    import com.jcabi.log.Logger;
public class Foo {
  public void save(String file) {
    // save the file
    Logger.info(this, "file %s saved successfuly", file);
  }
}

    Это выглядит намного более чистым для меня и делает точно то же самое - отправляет одну строку журнала в систему регистрации SLF4J. Кроме того, он автоматически проверяет, включен ли заданный уровень регистрации (для оптимизации производительности) и форматирует заданную строку с использованием Formatter (то же самое, что и String.format()).

    Для удобства, в библиотеке также реализовано несколько “декораторов”.

    Библиотека поставляется в качестве зависимости JAR в Maven Central (получите ее последние версии в Maven Central).

    <dependency>
  <groupId>com.jcabi</groupId>
  <artifactId>jcabi-log</artifactId>
</dependency>
