    An Empty Line is a Code Smell

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Тема может показаться шуткой, но это не так. Пустая строка, используемая в качестве разделителя инструкций в методе объекта, является “запахом кода” (code smell). Почему? Коротко говоря, потому что метод не должен содержать “части”. Метод всегда должен делать одну вещь, и его функциональное разделение должно осуществляться с помощью конструкций языка (например, новых методов), а никогда пустыми строками.

    Посмотрите на этот класс Java (он воняет, не так ли?):

    final class TextFile {
  private final File file;
  TextFile(File src) {
    this.file = src;
  }
  public int grep(Pattern regex) throws IOException {
    Collection<String> lines = new LinkedList<>();
    try (BufferedReader reader =
      new BufferedReader(new FileReader(this.file))) {
      while (true) {
        String line = reader.readLine();
        if (line == null) {
          break;
        }
        lines.add(line);
      }
    }

int total = 0;
for (String line : lines) {
  if (regex.matcher(line).matches()) {
    ++total;
  }
}
return total;
 } } ```

Этот метод первым делом загружает содержимое файла. Во-вторых, он подсчитывает, сколько строк соответствуют заданному регулярному выражению. Итак, почему метод `grep` плохо пахнет? Потому что он делает две вещи вместо одной --- он загружает и выполняет поиск.

Если мы установим правило, чтобы избегать пустых строк в телах методов, то метод придется переработать, чтобы сохранить "разделение обязанностей", введенное этой пустой строкой.

    final class TextFile { private final File file; TextFile(File src) { this.file = src; } public int grep(Pattern regex) throws IOException { return this.count(this.lines(), regex); } private int count(Iterable lines, Pattern regex) { int total = 0; for (String line : lines) { if (regex.matcher(line).matches()) { ++total; } } return total; } private Iterable lines() throws IOException { Collection lines = new LinkedList<>(); try (BufferedReader reader = new BufferedReader(new FileReader(this.file))) { while (true) { String line = reader.readLine(); if (line == null) { break; } lines.add(line); } return lines; } } }

    
Я считаю, что очевидно, что у этого нового класса методы намного более связаны и читаемы. Теперь каждый метод делает ровно одно действие, и легко понять, какое именно действие.

Эта идея об избегании пустых строк также применима и к другим языкам, не только к Java/C++/Ruby и т.д. Например, этот CSS код явно требует рефакторинга:

```css
.container {
  width: 80%;
  margin-left: auto;
  margin-right: auto;

font-size: 2em; font-weight: bold; } ```

Пустая строка здесь говорит нам (и на самом деле кричит на нас), что класс `.container` слишком сложен и должен быть разложен на два класса:

    .wide { width: 80%; margin-left: auto; margin-right: auto; } .important { font-size: 2em; font-weight: bold; } ```

    К сожалению, использование пустых строк для разделения блоков кода является очень распространенной привычкой. Более того, часто я вижу пустые блоки из двух или даже трех строк, которые все выполняют эту негативную роль разделителя задач.

    Не нужно говорить, что должно быть только несколько публичных методов в правильно разработанном классе, и правильно разработанный метод должен содержать не более десяти инструкций (согласно Бобу Мартину). Пустые строки внутри методов побуждают нас нарушать эту замечательную правило и превращать их в многостраничные поэмы.

    Конечно, проще просто несколько раз нажать enter и продолжать писать код в том же методе, не задумываясь и не проводя рефакторинг. Эта лень в конечном итоге приведет к коду, который вообще трудно поддерживать.

    Чтобы предотвратить подобное в своих проектах, полностью прекратите использование пустых строк внутри методов. Идеально было бы запретить их в автоматизированной сборке. В qulice.com, инструменте статического анализа, которым мы пользуемся во всех Java-проектах, мы создали собственную проверку Checkstyle, которая запрещает пустые строки в каждом методе.
