    Xembly, an Assembly for XML

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Я использую XML практически в каждом своем проекте. И, несмотря на всю суматоху вокруг JSON/YAML, я искренне верю, что XML - один из самых великих языков, когда-либо созданных. Кроме того, я считаю, что красота XML проявляется, когда он используется в сочетании с сопутствующими технологиями.

    Например, вы можете предоставить свои данные в формате XML и отобразить их для конечного пользователя, используя стиль XSL.

    Еще одним примером будет проверка одних и тех же данных перед отображением, чтобы убедиться в правильности структуры. Вы можете сделать это с помощью схемы XSD. В качестве альтернативы, вы можете выбрать конкретные элементы данных из всего документа, используя запросы XPath.

    По сути, эти три технологии - XSL, схема XSD и XPath - и делают XML таким мощным.

    Однако иногда XML может быть недостаточным. К примеру, представьте, что у вас есть существующий документ, который требуется незначительно изменить. Например, давайте используем следующее:

    <accounts>
  [...]
  <acc id='34'>
    <name>Jeffrey</name>
    <balance>305</balance>
  </acc>
  <acc id='35'>
    <name>Walter</name>
    <balance>50090</balance>
  </acc>
  [...]
</accounts>

    Вышеуказанный код представляет собой список аккаунтов. Каждый аккаунт имеет свой собственный id и несколько дочерних элементов. В нашем примере нам нужно найти аккаунт, принадлежащий Jeffrey, и увеличить его баланс на 500. Как мы это сделаем?

    Ну, существует несколько возможных решений:

    • SAX-разберите документ, измените баланс и сохраните поток;

    • DOM-разберите его, найдите элемент с помощью XPath, измените значение и затем напечатайте его.

    • применить параметризованную таблицу стилей XSL;

    • применить небольшой скрипт XQuery для внесения изменений

    Все эти методы имеют свои недостатки. Однако, у них всех есть одна общая проблема - они очень громоздкие. Для каждого из вышеуказанных методов вам понадобится как минимум одна страница кода, чтобы выполнить эту довольно простую операцию. Более того, если логика операции становится более сложной, количество необходимого кода растет намного быстрее, чем вы можете ожидать.

    Проще говоря, XML не имеет инструмента для примитивной манипуляции данными внутри документа. Возможно, именно эта недостаточность делает XML непопулярным среди некоторых пользователей.

    В любом случае, вот инструмент, который я создал несколько месяцев назад: Xembly. Это императивный язык с несколькими простыми инструкциями, который по стилю напоминает Assembly. Именно поэтому он и называется Xembly. В Xembly нет циклов, условий или переменных - только последовательность инструкций с аргументами.

    Давайте создадим простой пример. Предположим, например, что мы хотим добавить новый номер счета 36 в наш документ со списком. Код будет выглядеть так:

    XPATH '/accounts';
ADD 'account';
ATTR 'id', '36';
ADD 'name';
SET 'Donny';
UP;
ADD 'balance';
SET '3400';

    Вышеуказанное должно быть интуитивно понятно, но я объясню на всякий случай. Сначала директива XPATH указывает нам на элемент, найденный с помощью запроса XPath “/accounts”. Это будет наш корневой элемент. Мы предполагаем, что он существует в документе. Поэтому, если его нет, наш сценарий Xembly завершится с исключением времени выполнения.

    Далее, директива ADD в строке 2 создает новый XML-элемент без детей или атрибутов. Затем директива ATTR задает атрибут для этого элемента. Затем код добавляет новый дочерний элемент name и устанавливает его текстовое значение на "Donny" с помощью директивы SET. Наконец, мы перемещаем указатель обратно к элементу account с помощью UP, добавляем дочерний элемент balance и устанавливаем его значение на "3400".

    Наша задача изменения баланса может быть выражена в Xembly с помощью следующего кода:

    XPATH '/accounts/account[name="Jeffrey"]/balance';
XSET '. + 500';

    Директива XSET устанавливает значение текста элемента, аналогично SET, но предварительно вычисляет его, используя предоставленное выражение XPath . + 500.

    Xembly выполняет все манипуляции через DOM. Следовательно, Xembly может быть реализован внутри любого языка, который имеет встроенную реализацию DOM.

    В настоящее время существует только одна реализация языка Xembly - на языке Java. Вот как это работает:

    Iterable<Directive> directives = new Directives()
  .xpath("/accounts")
  .add("account")
  .attr("id", "36")
  .add("name").set("Donny").up()
  .add("balance").set("3400");
new Xembler(directives).apply(document);

    В этом отрывке я использую дополнительный конструктор сценариев Directives, который позволяет создавать директивы в плавном стиле. Затем я использую класс Xembler, который похож на “ассемблер”, чтобы применить все указанные директивы к объекту document класса org.w3c.dom.Document.

    Кроме того, Xembly может использоваться для создания XML-документов с нуля и в качестве замены традиционного построения DOM. Быстрый пример:

    System.out.println(
  new Xembler(
    new Directives().add("html")
      .add("head")
      .add("title")
      .set("Hello, world!")
  ).xml()
);

    Вышеуказанный отрывок создает следующий вывод:

    <html>
  <head>
    <title>Hello, world!</title>
  </head>
</html>

    Для меня это кажется более простым и компактным.
