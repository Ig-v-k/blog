    Abstract Objects

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Как создавать объекты в вашем объектно-ориентированном языке? Давайте возьмем что-то классическое, например C++, Java или C#. Сначала вы определяете класс, а затем создаете экземпляр этого класса. Первый шаг известен как абстракция, а второй - инстанциация. В функциональном программировании существует аналогичная пара операций: объявление функции - это абстракция, а вызов функции с конкретными аргументами - аппликация. Вопрос заключается в следующем: почему ООП нуждается в классах и объектах, в то время как ФП обходится только функциями?

    Вот абстрактный объект на EO:

    [id db] > book
  db.query > title
    "SELECT title FROM book WHERE id=?"
    id

    Название объекта - book. У него есть три атрибута: id, db и title. Первые два являются «свободными»: они еще не привязаны ни к одному объекту. Третий атрибут title уже привязан к копии объекта db.query. Объект book является абстрактным, потому что у него есть свободные атрибуты - он не достаточно конкретен, чтобы ссылаться на конкретную сущность в реальном мире.

    Конструкция db.query берет абстрактный объект query из объекта, привязанного к атрибуту db, и создает его копию, указывая два аргумента: SQL-строку и id. Процесс копирования абстрактного объекта называется «применением», поскольку он очень похож на применение аргументов к функции в ФП.

    Правильный способ использования абстрактного book - создать его копию, указав аргументы:

    book 42 mysql > b

    Здесь 42 - это id, mysql - это db, а b - новый объект, копия book. В 𝜑-исчислении, которое является основой языка программирования EO, это будет записано следующей формулой:

    b ↤ book(42, mysql)

    Чтобы получить title из b и назвать его t, мы просто делаем:

    b.title > t

    Мы также можем объединить все это в одну строку:

    (book 42 mysql).title > t

    Возможно применить некоторые аргументы к абстрактному объекту, оставив другие атрибуты свободными. Например, это сделает другой абстрактный объект, который все еще требует указания id, чтобы стать закрытым объектом.

    book mysql:db > x

    Суффикс :db помогает компилятору понять, какой свободный атрибут должен быть привязан к объекту mysql. Позже x может быть скопирован снова, и будет создан закрытый объект b.

    x 42 > b

    Несмотря на то, что синтаксис EO позволяет использовать book.title, это приведет к ошибке времени компиляции: запрещено обращаться к свободным атрибутам абстрактного объекта или атрибутам, которые зависят от других свободных атрибутов. Однако предположим, что мы добавляем новый атрибут print к book, чтобы вывести его id в консоль, мы должны сделать следующее:

    [id db] > book
  stdout > print
    sprintf
      "The book ID is %d"
      id
  db.query > title
    "SELECT title FROM book WHERE id=?"
    id

    Атрибут print не требует привязки атрибута db, поскольку он не использует его. Возможно скопировать book только с id и затем вызвать print (предполагая, что это происходит в интерактивной консоли EO).

    $ book 42:id > x
x
$ x.print
The book ID is 42
