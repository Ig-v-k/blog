    Wring.io, a Dispatcher of GitHub Notifications

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Я участвую в более чем 50 репозиториях на GitHub. Мы управляем всеми нашими проектами там. GitHub присылает мне сотни писем каждый день. Я серьезно. Сотни! Я попытался как-то отфильтровать их в Gmail, но это не совсем возможно. Фильтры Gmail недостаточно мощные, чтобы понять разницу между разными типами уведомлений, и есть много других проблем. Я решил создать свою собственную простую машину для фильтрации. Она называется wring.io.

    Идея wring.io проста. Сначала я регистрирую свои источники уведомлений (называемые “pipes”), такие как GitHub. Затем я предоставляю wring.io разрешение подключаться к GitHub от моего имени и получать все новое оттуда.

    Затем я настраиваю, что должно быть отфильтровано, используя сопоставление текста и/или регулярные выражения. Сразу после создания нового канала, wring.io начинает собирать все мои источники и обновлять мою входящую почту. Все, что мне нужно сделать, это удалить новые сообщения из своей входящей почты, когда я закончил с ними. Вот и все.

    Давайте рассмотрим пример. Сначала я создаю новый pipe:

    Это объект JSON. Свойство class должно быть установлено на io.wring.agents.github.AgGithub. Это имя Java-класса, который будет получать мои уведомления с GitHub. Проект является открытым исходным кодом, поэтому вы можете увидеть, как на самом деле работает этот класс: AgGithub.

    Свойство token должно быть установлено на персональный токен доступа, который я должен создать сначала в GitHub. Сервер будет подключаться к GitHub от моего имени и с использованием моих учетных данных.

    Свойство ignore должно иметь массив строк. Каждый элемент - это сопоставляющий шаблон. Можно использовать текст или регулярное выражение. По умолчанию используется текст. Если в уведомлении найден точно такой же текст, он будет проигнорирован. Чтобы использовать регулярное выражение, его нужно заключить в косые черты (например /[a-z]+/). Вы можете пропустить это свойство и просто указать этот JSON.

    {
  "class": "io.wring.agents.github.AgGithub",
  "token": "your-personal-access-token"
}

    Затем я открываю свою почту и читаю то, что там есть.

    Это решение буквально экономит мне часы времени сейчас. Не стесняйтесь использовать его, оно абсолютно бесплатно. Более того, оно открытое исходное код, так что не стесняйтесь вносить свой вклад.
