    ReHTTP.net—HTTP Repeater

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Несколько недель назад у меня возникла проблема с 0pdd.com - одним из моих веб-приложений, которое должно принимать HTTP-запросы (известные как webhooks) от GitHub: иногда приложение перестает работать, GitHub получает ошибку HTTP и больше не отправляет запрос. Запрос просто теряется. После восстановления работы приложения невозможно получить его повторно. Я понял, что мне нужна сетка сервисов между GitHub и моим веб-приложением, чтобы принимать HTTP-запросы и повторять их позже, если они не могут быть доставлены сразу.

    Я создал rehttp.net, чтобы именно это делать.

    Понимаете, URL-адрес, который я указывал в GitHub, - это следующий:

    http://www.0pdd.com/hook/github

    С этого момента необходимо использовать новый URL.

    https://www.rehttp.net/p/http://www.0pdd.com/hook/github

    Он выглядит очень похоже, но начинается с https://www.rehttp.net/p/. GitHub отправляет все запросы с методами PUT/POST вебхука на сервер ReHTTP, который сохраняет их во временной базе данных (я использую AWS DynamoDB).

    ReHTTP пытается доставить их немедленно. Если что-то идет не так и код ответа сервера не находится в диапазоне 200-299, следующая попытка делается примерно через час. Затем повторные попытки осуществляются каждый час в течение примерно одного дня. Если все попытки проваливаются, запрос отменяется и все.

    Интересно, что теперь я могу видеть сводку моего API здесь. Я вижу, сколько запросов было успешно обработано за последние 24 часа и сколько из них завершились неудачей. Кроме того, у меня есть этот милый кнопочка:

    И у меня есть URL для проверки статуса всего API:

    https://www.rehttp.net/s?u=http%3A%2F%2Fwww.0pdd.com%2Fhook%2Fgithub

    Я передал этот URL-адрес StatusCake, чтобы он отправлял запросы каждые пять минут. Если что-то пойдет не так, StatusCake отправит мне электронное письмо и сообщение на телефон.

    ReHTTP абсолютно бесплатен. Он написан на Java и его код открытый. Посмотрите его репозиторий на GitHub и внесите свой вклад, если вы найдете какие-либо ошибки или просто хотите добавить функцию.
