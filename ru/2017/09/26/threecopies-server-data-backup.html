    ThreeCopies.com—Server-Side Data Backup Service

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    У меня есть несколько ресурсов с данными, которые находятся только в одном месте, и которые я действительно не хочу потерять. Например, у меня есть веб-сайт на PHP с хостингом, и база данных MySQL, также находящаяся в этом месте. У меня также есть база данных NoSQL в Amazon DynamoDB, база данных PostgreSQL на Heroku, и еще… Ну, их много. Как их резервировать всегда было для меня вопросом.

    Наиболее простым способом является аренда недорогого сервера за $15/месяц (или использование уже имеющегося) и настройка Cron для запуска пользовательского bash-скрипта, который будет извлекать данные из базы данных MySQL, упаковывать их и загружать в надежное место, такое как Amazon S3 bucket. Затем мне понадобится еще один скрипт для базы данных PostgreSQL и еще один для FTP-архива файлов и т.д.

    На самом деле, я так делал уже много лет. Недостатки этого решения всегда были одинаковыми:

    • Мне нужно было заплатить за сервер.

    • Мне было необходимо убедиться, что сервер всегда работает (Linux далеко не надежна).

    • Мне также потребовалось сделать резервную копию моих скриптов.

    • Мне приходилось каждый раз подключаться к серверу по SSH, когда я хотел изменить скрипт, помнить, где они находятся, как они запускаются и так далее.

    Самая большая проблема заключается в том, что каждый владелец источника данных сталкивается с точно таким же набором проблем. “Почему я не могу создать хостинговое решение для этих скриптов, чтобы помочь всем сохранить свои данные,” я задавал себе этот вопрос много лет. “Ну, я могу,” был ответ всего несколько недель назад, и я создал ThreeCopies.

    Это очень простой размещенный исполнитель bash-скриптов, которые вы редактируете через веб-интерфейс. Затем один из наших серверов запускает контейнер Docker (yegor256/threecopies - это образ, вот Dockerfile) и выполняет ваш скрипт внутри него.

    Сценарий запускается каждый час, каждый день и каждую неделю. Отсюда и название: “три копии”. Хорошей практикой для резервного копирования данных является создание отдельных копий с разными регулярностями. Кроме того, вы можете хотеть помещать разные данные в разные копии. Чтобы помочь вашему сценарию понять, какая копия запущена в данный момент, мы передаем в него переменную окружения $period со значением hour, day или week.

    Как ваш скрипт извлекает данные, упаковывает и архивирует их, зависит от данных. Я создал краткое руководство для наиболее распространенных сценариев. Например, вот как я создаю резервную копию базы данных MySQL для thePMP:

    ```bash

    I don’t want to back up every hour

    if [ “${period}” == “hour” ]; then exit 0; fi

    I dump the entire database into the file

    mysqldump –lock-tables=false –host=db.thepmp.com \ –user=thepmp –password=***** \ –databases thepmp > thepmp.sql

    I compress the file

    tgz=”$(date “+%Y-%m-%d-%H-%M”).tgz” tar czf “${tgz}” thepmp.sql

    I upload it to Amazon S3 bucket

    echo “[default]” > ~/.s3cfg echo “accesskey=AKIAICJKH**CVLAFA” » ~/.s3cfg echo “secretkey=yQv3g3ao654Ns***H1xQSfZlTkseA0haG” » ~/.s3cfg s3cmd –no-progress put “${tgz}” “s3://backup.yegor256.com/${tgz}” ```

    Вывод скрипта доступен через веб-интерфейс, и это еще одно преимущество данного решения. Легко отслеживать, что пошло не так, и перезапустить скрипт. Все журналы доступны через браузер. Нет SSH, нет терминалов.

    Я бы сказал, что это легкая версия AWS Data Pipeline. ThreeCopies делает точно то же самое, но его легче настроить и дешевле. Я взимаю 0,01 доллара за каждый час выполнения скрипта. И я фактически взимаю плату за секунды, в то время как AWS всегда взимает плату за полный час. За 5,00 долларов вы получаете 500 часов. Например, скрипт, который вы видите выше, занимает около 5 минут для завершения (база данных не очень большая). Если вы пропустите часовые выполнения, как я сделал выше, вы будете использовать 170 минут серверного времени каждый месяц, что будет стоить вам примерно 0,34 доллара в год! Я считаю, что это намного дешевле, чем сервер и его мониторинг.

    Ещё одна вещь перед тем, как вы уйдете. ThreeCopies написан на Java 8 и является открытым исходным кодом. Вы можете найти его на GitHub. Не стесняйтесь проверять код, находить ошибки и вносить исправления или улучшения.
