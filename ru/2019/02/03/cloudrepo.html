    How to Deploy Maven Artifacts to CloudRepo via Rultor

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    В моей предыдущей статье я описал, как настроить частный репозиторий Maven в Amazon S3 и развернуть там через Rultor. Это отличное решение, если вы знакомы с управлением Amazon Web Services (AWS), S3 и AWS Identity and Access Management (IAM). Однако, если вам неудобно управлять учетной записью AWS и всеми связанными разрешениями, вам может понадобиться хранить ваши артефакты Apache Maven в облачном менеджере репозиториев. Вот как настроить Rultor для развертывания ваших зависимостей Maven в CloudRepo. Я написал этот блог пост вместе с Крисом Шелленбаргером, их основателем.

    И менеджеры репозиториев, и S3 позволяют вашим инструментам сборки сохранять и получать библиотеки программного обеспечения в удаленном репозитории или корзине. Однако менеджеры репозиториев берут на себя много работы, которую вам пришлось бы управлять самостоятельно с помощью решения S3.

    Хотя AWS достаточно надежен и может быть настроен для выполнения всех функций менеджера репозиториев, есть случаи использования, которые работают прямо “из коробки” при выборе полностью управляемого решения, такие как: управление пользователями/группами, специфические представления Maven, вебхуки, уведомления, доступ и журналы аудита, полностью управляемая безопасность и т. д. Простыми словами, это не просто хранилища JAR-файлов, а Maven-репозитории в облаке.

    Полностью управляемое облачное решение будет стоить вам ежемесячную плату за подписку. На момент написания они взимают $9 в месяц и больше. Они бесплатны для открытых исходных кодов, но я не вижу причин, почему бы вам не использовать Maven Central. Хорошая новость заключается в том, что они предлагают специальную скидку для моих читателей: первые 90 дней бесплатны, вместо стандартных 14. Это фактически скидка 25% на первый год. Просто отправьте им письмо, когда ваш аккаунт будет создан, и они позаботятся о вас (не забудьте упомянуть меня, чтобы получить скидку).

    Предположим, что вы создали учетную запись в CloudRepo и настроили как пользователя, так и репозиторий Maven, для развертывания в CloudRepo требуется выполнить два шага на стороне клиента: 1) Настроить файл settings.xml с учетными данными и 2) Добавить ваш репозиторий в раздел <distributionManagement> файла pom.xml.

    Местоположение файла settings.xml по умолчанию находится в вашем каталоге ~/.m2. Откройте этот файл и убедитесь, что в нем есть запись <server>, как показано ниже:

    <settings>
  <servers>
    <server>
      <id>io.cloudrepo</id>
      <username>yegor256@gmail.com</username>
      <password>my-secret</password>
    </server>
  </servers>
</settings>

    Объявляя server и указывая id, вы можете ссылаться на id внутри файлов Maven POM. Когда Maven пытается аутентифицироваться на сервере, он будет искать соответствующий ключ в файле settings.xml.

    Если вы беспокоитесь о хранении вашего пароля в виде обычного текста в файловой системе, проверьте, как Maven рекомендует исправить это.

    Теперь, когда ваши учетные данные установлены, вам необходимо указать свой pom.xml на ваш репозиторий CloudRepo. Добавьте новый элемент <repository> в раздел <distributionManagement> (для загрузки) вашего pom.xml и в раздел <repositories> (для загрузки).

    <project>
  [...]
  <repositories>
    <repository>
      <id>io.cloudrepo</id>
      <url>https://[your-org-name].mycloudrepo.io/repositories/[your-repository-name]</url>
    </repository>
  </repositories>
  [...]
  <distributionManagement>
    <repository>
      <id>io.cloudrepo</id>
      <name>CloudRepo Maven Repository</name>
      <url>https://[your-org-name].mycloudrepo.io/repositories/[your-repository-name]</url>
    </repository>
  </distributionManagement>
</project>

    Когда у вас есть рабочая сборка в локальной среде, вам нужно развернуть ее в продакшн с помощью Rultor. Сначала вам нужно сохранить ваши учетные данные в Git, чтобы Rultor мог получить к ним доступ во время сборки и развертывания. Однако очень важно никогда не хранить ваши учетные данные в открытом виде в системе контроля версий (замените имя вашего проекта GitHub на my/project).

    $ gem install rultor
$ rultor encrypt -p my/project settings.xml

    Это создает зашифрованную версию вашего файла настроек с именем settings.xml.asc. Добавьте этот файл в корневую директорию вашего проекта, затем выполните коммит и пуш. Это безопасно. Только Rultor имеет ключи для расшифровки этого файла, поэтому даже если ваш исходный код будет доступен другим, ваши учетные данные будут сохранены в безопасности.

    Чтобы включить Rultor, добавьте файл .rultor.yml в корневой каталог вашего проекта со следующим содержимым:

    decrypt:
  settings.xml: "repo/settings.xml.asc"
deploy:
  script: |
    mvn clean deploy --settings ../settings.xml

    Для получения дополнительной информации о файле .rultor.yml просмотрите страницу справки Rultor.

    Теперь, когда все настроено, вы должны иметь возможность развернуть в CloudRepo с помощью Rultor, выполнив команду @rultor deploy. Дождитесь ответа, и Rultor позаботится обо всем остальном.

    That’s it!
