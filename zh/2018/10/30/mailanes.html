    Mailanes.com Helps Manage Newsletters and Mailing Lists

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    还有其他类似的产品，包括MailChimp、AWeber等等。它们挺好的，也许吧，我不太确定。我以前试过其中一些，但感到很沮丧。它们既昂贵又笨重。这是我的经验。我想要一个轻量级的东西，可以上传邮箱列表，创建简单的HTML/纯文本邮件，配置我的SMTP出站服务器，然后就可以了。我花了几天时间，创建了Mailanes。它对所有人都是免费的，简单易用，并且是开源的。

    首先，您通过GitHub登录（当然，还有其他方式，对吧？）然后您创建一个收件人列表。您可以逐个添加他们，或者上传一个CSV文件。接下来，您创建一个信件通道。通道是您的收件人应该一步一步通过的路径，一封一封的电子邮件。然后，您创建一个活动并将您的列表与通道连接起来。就是这样。

    让我通过示例来展示。假设你访问 zerocracy.com，输入你的电子邮件并点击“帮助我开始！”网站上的表单已配置为直接将该电子邮件发送到Mailanes网络应用程序（参见来源）。

    <form method="POST" action="https://www.mailanes.com/subscribe?list=4">
  <input type="hidden" name="source" value="www.zerocracy.com"/>
  <input type="hidden" name="redirect" value="https://www.zerocracy.com/"/>
  <input type="email" name="email" placeholder="Your email..." size="24"/>
  <button type="submit">Help me start!</button>
</form>

    表单的action指向Mailanes服务器上的/subscribe?list=4 URL。一旦提交，收件人将被添加到第4个列表中，并且浏览器将被重定向到redirect表单参数中的URL。

    在收件人出现在列表中后，系统会开始按照车道中定义的字母顺序将其移动。您可以自行尝试，您几乎可以立即收到第一封信（准确来说，需时10分钟）。

    为了定义一封信，我决定使用 Liquid，这是一种相当简单的标记语言。对于列表、通道、信件和活动的配置，我使用 YAML，因为它比 JSON 和 XML 更简单、更短，但又足够强大。以下是你将收到的第一封信的配置示例：

    信的 relax 参数指定了系统在发送下一封信之前需要等待的时间。"4:0:0" 表示四天零小时零分钟。

    在信的底部我添加了这个：

    Thanks,<br/>
Yegor Bugayenko<br/>
Click [here]( unsubscribe }}) if you don't want to hear from me again.

    unsubscribe }} 将被替换为一个链接，该链接指向一个页面，收件人可以立即取消订阅电子邮件。我相信允许你的通讯订阅者立即取消订阅是一个好的做法。他们还可以通过 List-Unsubscribe 邮件头来执行取消订阅操作。

    我还配置了使用我的SMTP服务器的详细信息。我正在使用AWS 简易邮件服务。这个服务配置起来非常简单，而且价格便宜：每10,000封邮件1美元（截至目前）。据我所知，SendGrid的价格是它的两倍，Smtp2Go的价格是它的六倍，SendInBlue的价格是它的四倍，MailGun的价格是它的八倍，等等。

    然后，我配置了我的列表，通过Telegram通知我有关所有新订阅者、取消订阅等操作的信息。

    你看，这里有一个 Telegram 聊天 ID，我会在那里收到通知。你也可以这样做，你只需要知道聊天 ID。开始与 Telegram 机器人 @mailanes_bot 对话，它会告诉你你的聊天 ID。

    我也在我的博客通讯中使用Mailanes。试试在这里订阅，你会看到会发生什么。

    README 全面解释了所有可能的配置选项。该项目是开源的，因此请随时通过拉取请求进行贡献。正如我上面所说，对任何人来说都是完全免费的。只需注册，创建您的列表、通道和活动，然后开始使用。

    如果系统中有什么不清楚的地方，让你感到迷茫，请毫不犹豫地提交工单，我会尽力改进或添加文档。
