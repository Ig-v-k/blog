    Abstract Objects

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    你是如何在你的面向对象语言中创建对象的呢？让我们以一些经典的语言为例，比如C++、Java或C#。首先，你要定义一个类，然后创建一个实例。第一步被称为抽象，第二步被称为实例化。在函数式编程中也存在类似的一对操作：声明函数是抽象，而使用特定参数调用函数是应用。问题是：为什么面向对象编程需要类和对象，而函数式编程只需要函数就能生存下去？

    这是EO中的一个抽象对象：

    [id db] > book
  db.query > title
    "SELECT title FROM book WHERE id=?"
    id

    对象的名称是 book。它有三个属性：id、db 和 title。前两个属性是“自由的”：它们尚未绑定到任何对象。第三个属性 title 已经绑定到对象 db.query 的副本上。对象 book 是抽象的，因为它有一些自由属性——它不足以指代现实世界中的特定实体。

    构造函数 db.query 从绑定到属性 db 的对象中取出抽象对象 query 并复制它，同时指定两个参数：SQL 字符串和 id。复制抽象对象的过程称为“应用”，因为它与函数式编程中对函数应用参数的过程非常相似。

    使用抽象的 book 的正确方式是复制它，并指定参数：

    book 42 mysql > b

    这里，42是id，mysql是db，而b是一个新对象—book的一个副本。在𝜑-计算中，这可以表示为以下公式：

    b ↤ book(42, mysql)

    为了获取b的title并将其命名为t，我们只需要执行以下操作：

    b.title > t

    我们也可以将所有内容放在一行中。

    (book 42 mysql).title > t

    可以对一个抽象对象应用一些参数，同时保留其他属性的自由。例如，这将创建另一个抽象对象，仍然需要指定id才能成为一个闭合对象。

    book mysql:db > x

    后缀:db帮助编译器理解应将哪个自由属性绑定到对象mysql。稍后，x可以再次复制，将创建一个闭合对象b。

    x 42 > b

    尽管EO的语法允许使用book.title这样的语句，但这会导致编译时错误：禁止访问抽象对象的自由属性或依赖其他自由属性的属性。然而，假设我们给book添加一个新属性print，为了将其id打印到控制台，我们可以按照以下步骤进行操作：

    [id db] > book
  stdout > print
    sprintf
      "The book ID is %d"
      id
  db.query > title
    "SELECT title FROM book WHERE id=?"
    id

    属性print不需要绑定db属性，因为它不使用它。可以仅使用id复制book，然后调用print（假设在EO交互式控制台中进行）：

    $ book 42:id > x
x
$ x.print
The book ID is 42

    顺便问一下，你知道还有其他带有抽象对象的语言吗？还是EO是第一个引入这种设计的？
