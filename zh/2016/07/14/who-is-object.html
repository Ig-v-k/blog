    Who Is an Object?

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    有成千上万本关于面向对象编程的书籍，数以百计的面向对象语言，我相信大多数（读作“全部”）都给出了一个错误的“对象”定义。这就是为什么整个面向对象编程世界充斥着误解和错误的原因。他们对对象的定义受限于他们所使用的硬件架构，因此非常原始和机械。我想介绍一个更好的定义。

    什么是对象？我做了一些研究，这是我找到的内容：

    • 对象可能包含数据，以字段的形式出现，通常被称为属性；以过程的形式表示的代码，通常被称为方法。

    • 一个对象通过字段存储其状态，并通过方法暴露其行为。

    • 每个对象看起来很像一个小型计算机—它具有状态，并且具有可以要求其执行的操作。

    • 一个类是一组包含值的数据字段和对这些值进行操作的方法。

    • 一个对象是一些内存，它存储了某种类型的值。

    • 一个对象由一些私有内存和一组操作组成。

    所有这些定义中共同的是“包含”（或“持有”，“由…组成”，“具有”等）这个词。他们都认为对象是一个带有数据的盒子。而这个观点正是我坚决反对的。

    如果我们看一下C++或Java的实现方式，这样对于对象的定义将技术上是正确的。确实，对于每个对象，Java虚拟机在内存中分配了一些字节以存储对象的属性。因此，从技术上讲，在那种语言中，我们可以说对象是一个带有数据的内存盒子。

    没错，但这只是一个特殊情况！

    让我们试着想象另一种不在内存中存储对象属性的面向对象语言。困惑吗？请耐心等待一分钟。假设在这种语言中我们定义了一个对象：

    c {
  vin: v,
  engine: e
}

    在这里，vin和engine是对象c的属性（它是一辆车；现在先忘记类，专注于对象）。因此，有一个简单的对象具有两个属性。第一个属性是汽车的VIN，第二个属性是它的引擎。VIN是一个对象v，而引擎是e。为了更容易理解，这是一个类似对象在Java中的样子：

    char[] v = {'W','D','B','H',...'7','2','8','8'}; // 17 chars
Engine e = new Engine();
Car c = new Car(v, e);

    我对JVM不太确定，但在C++中，这样一个对象在内存中占用确切的25个字节（假设是64位x86架构）。前17个字节将被字符数组占用，另外8个字节将被指向内存块中的对象e的指针占用。这就是C++编译器理解对象并将其转换为x86架构的方式。在C++中，对象只是具有明确定义的数据属性分配的数据结构。

    在该示例中，属性 vin 和 engine 不相等：vin 是 “data”，而 engine 是指向另一个对象的 “pointer”。我故意这样做是为了证明只有 vin 才能将一个对象称为带有数据的盒子。只有当数据实际上位于对象的 “内部” 时，我们才能说该对象实际上是数据的容器。对于 engine 来说，这并不是真的，因为从技术上讲，对象内部没有数据。相反，它是指向另一个对象的 “pointer”。如果我们的对象只有一个 engine 属性，它只需要占用 8 个字节的内存，其中没有一个实际上被 “data” 占用。

    现在，让我们回到我们的新伪语言。让我们想象它与C++非常不同- 它根本不在内存中保存对象属性。它没有指针，也不知道任何关于x86架构的东西。它只是某种方式知道哪些属性属于一个对象。

    因此，在我们的语言中，对象不再是从技术和概念上都是数据盒子。它们知道数据在哪里，但它们并不“包含”数据。它们“表示”数据，以及其他对象和实体。事实上，在我们的虚构语言中，对象c代表另外两个对象：一个VIN和一个引擎。

    总结一下，我们必须明白，虽然当前市场上大多数编程语言中机械定义一个对象的方式是正确的，但是在概念上是非常不正确的，因为它将一个对象视为一个盒子，其中的数据对外界来说太过可见。这种可见性会引发我们以过程化的方式思考，并尽可能地访问这些数据。

    如果我们将对象视为数据的代表而不是其容器，我们就不会想尽快获取数据。我们会理解到数据是远在天边，我们不能轻易触及它们。我们应该与对象进行交流，而它与数据的交流方式并不是我们所关心的。

    我希望在不久的将来，市场将推出一种新的面向对象语言，即使在技术上也不会将对象存储为内存数据结构。

    顺便提一下，以下是我最喜欢的书《Object Thinking》（David West著）第66页中关于对象的定义：

    一个对象是构建宇宙的基本单位 “quanta” 的等效物。

    你觉得呢？这个定义是否接近我刚刚提出的“代表性”定义？
