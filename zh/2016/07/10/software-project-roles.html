    Key Roles in a Software Project

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我相信在大多数软件项目中应该有几个角色。根据Zerocracy的XDSD原则进行管理，我们在我们的项目中都有这些角色。然而，请注意在其他管理方法中，这些角色可能具有不同的含义。这篇博客文章主要是给与我们合作的人，不论是客户还是自由职业者。

    只有几个角色：

    项目经理（PM）负责保持项目处于控制之下。项目经理向我们的项目管理办公室（PMO）负责汇报。

    产品负责人 (PO) 是赞助商的代表。PO提供产品需求。PO提交错误报告并提出与其相关的任何顾虑或问题。通常，PO是一个非常熟悉产品工作原理并能够理解源代码的技术人员。

    软件架构师（ARC）负责整个技术解决方案。软件架构师被指责为所有技术问题负责。在我们可以合并之前，软件架构师必须批准所有拉取请求。软件架构师是项目中与产品负责人（PO）的主要联系人。软件架构师做出所有技术决策。软件架构师向项目经理（PM）报告。

    开发人员(DEV) 是一名程序员，负责关闭漏洞。开发人员向项目经理报告。

    需求分析师 (REQ) 负责对产品进行验证。REQ从PO那里征集需求。REQ向PO演示产品。验证失败并且产品需要更改时，REQ会提交新的错误报告。REQ向PM汇报工作。

    质量保证 (QA) 负责监督我们流程的正确性。在任务经由项目经理（PM）正式关闭之前，质量保证会批准每个已关闭的任务。质量保证确保我们的流程符合我们的政策。质量保证向项目经理汇报。

    Tester（TST）手动测试产品，找到漏洞并报告。

    除此之外，任何角色在发现错误时都被鼓励提交错误报告。
