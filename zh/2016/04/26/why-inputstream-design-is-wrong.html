    Why InputStream Design Is Wrong

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    这不仅仅是关于InputSteam，这个类是一个糟糕设计的好例子。我谈论的是三个重载方法read()。我在Elegant Objects的第2.9节中提到了这个问题。简而言之，我坚信接口应该是“功能贫乏的”。InputStream本应该首先是一个接口，它应该有一个单一的方法read(byte[])。然后，如果它的作者想要给我们额外的功能，他们应该创建补充的“智能”类。

    这是现在的样子：

    abstract class InputStream {
  int read();
  int read(byte[] buffer, int offset, int length);
  int read(byte[] buffer);
}

    怎么了？拥有读取单个字节、字节数组甚至是具有直接定位到缓冲区特定位置的字节数组的能力非常方便！

    然而，我们仍然缺少一些方法：用于将字节读取并立即保存到文件中的方法，将其转换为选定编码的文本的方法，通过电子邮件发送以及发布到Twitter上的方法。如果这些功能也能在这个可怜的InputStream中实现就太好了。我希望Oracle Java团队正在努力开发它们。

    同时，让我们看看这些聪明的工程师为我们设计的东西到底有什么问题。或者让我展示一下我会如何设计InputStream，然后我们来比较一下：

    interface InputStream {
  int read(byte[] buffer, int offset, int length);
}

    这是我的设计。InputStream 负责从流中读取字节。这个功能只有一个单一的方法。对每个人来说都方便吗？可以读取并发布到 Twitter 吗？还没有。我们需要那个功能吗？当然需要，但这并不意味着我们会将其添加到接口中。相反，我们将创建一个补充的“智能”类：

    interface InputStream {
  int read(byte[] buffer, int offset, int length);
  class Smart {
    private final InputStream origin;
    public Smart(InputStream stream) {
      this.origin = stream;
    }
    public int read() {
      final byte[] buffer = new byte[1];
      final int read = this.origin.read(buffer, 0, 1);
      final int result;
      if (read < 1) {
        result = -1;
      } else {
        result = buffer[0];
      }
      return result;
    }
  }
}

    现在，我们想要从流中读取一个字节。这是如何实现的：

    final InputStream input = new FileInputStream("/tmp/a.txt");
final byte b = new InputStream.Smart(input).read();

    读取单个字节的功能位于InputStream之外，因为这不是它的职责。流在读取数据后无需知道如何管理数据。流的责任仅限于读取，而不是解析或操作后续数据。

    接口必须小。

    显然，在接口中使用方法重载是一种代码异味。拥有超过三个方法的接口是重构的好候选对象。如果方法之间相互重载，那就是严重的问题。

    接口必须小！

    你可以说InputStream的创作者关心性能，这就是为什么允许我们以三种不同的方式实现read()。那么我不得不再次问，为什么不创建一个用于阅读并立即发布在Twitter上的方法呢？那将会非常快速。这不就是我们都想要的吗？一个快速的软件，没有人想要阅读或维护。

    Most likely, what is wrong with this Java interface? #elegantobjects #java

    interface Storage {
    int readInt();
    String readString();
    float readFloat();
    }
