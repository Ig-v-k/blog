    Don't Use Java Assertions

    在Java中，基本上有两种方法可以验证一个情况并在发生意外情况时进行投诉。它们要么是异常，要么是断言。从技术上讲，它们几乎是一样的，但存在一些细微差别。我认为在所有情况下都应该使用异常，而不应该使用断言。以下是原因。

    这是异常和断言之间的第一个区别。异常总是会被抛出，而断言默认情况下未启用。它们应该在测试期间打开，在生产环境中关闭。断言引发运行时异常AssertionError。但等等，它不是RuntimeException。它扩展了Error类，该类又扩展了Throwable。这是第二个区别。我不知道其他区别。

    我建议永远不要使用断言。这是因为我坚信采用“快速失败”方法。我认为错误不仅在测试期间可见，而且在生产环境中也应该可见。此外，我认为在生产环境中显露错误非常重要，如果您想要实现高质量的产品。

    因此，不要使用断言。它们只是Java（和其他一些语言）中一个有缺陷且过时的功能。
