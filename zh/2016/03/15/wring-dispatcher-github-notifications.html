    Wring.io, a Dispatcher of GitHub Notifications

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我在GitHub上参与了50多个仓库。我们在那里管理所有的项目。GitHub每天给我发送数百封电子邮件。我是认真的。数百封！我试着在Gmail中对它们进行一些过滤，但实际上并不可行。Gmail的过滤器不足以理解不同类型的通知之间的区别，而且还有许多其他问题。因此，我决定创建自己的简单过滤机器。它叫做wring.io。

    wring.io的理念很简单。首先，我会注册我的通知来源（称为“管道”），比如GitHub。然后，我会授权wring.io代表我连接到GitHub并获取那里的最新内容。

    然后我正在配置应该过滤掉的内容，使用文本匹配和/或正则表达式。在创建新的管道后，wring.io开始拉取我所有的来源并更新我的收件箱。当我处理完新消息后，我只需要从收件箱中删除它们。就是这样。

    让我们来看一个例子。首先，我创建一个新的管道：

    这是一个JSON对象。属性class必须设置为io.wring.agents.github.AgGithub。这是一个Java类的名称，将从GitHub获取我的通知。这个项目是开源的，所以你可以看到这个类的实际工作方式：AgGithub。

    属性token必须设置为我在GitHub上首先应该创建的个人访问令牌。服务器将代表我连接到GitHub，并使用我的凭据。

    属性ignore必须是一个字符串数组。每个项都是一个匹配模式。我可以使用文本或正则表达式。默认情况下，它是一个文本。如果在通知中找到完全相同的文本，则将被忽略。要使用正则表达式，我需要用斜杠包裹它（例如/[a-z]+/）。您可以跳过该属性，并只指定此JSON。

    {
  "class": "io.wring.agents.github.AgGithub",
  "token": "your-personal-access-token"
}

    然后我去收件箱看看里面有什么。

    这个解决方案现在真的为我节省了很多时间。随意使用它，它是完全免费的。此外，它是开源的，所以随意贡献。
