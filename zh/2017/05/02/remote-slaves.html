    A Remote Slave Is Still a Slave

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    远程工作显然是一个趋势，根据美国劳工统计局（BLS）和我的个人观察。”让他们在家工作”似乎是每两个初创企业和一些大公司（如Buffer，Automattic，Groove等）的万能解药，以及其他许多公司。然而，在大多数情况下，用虚拟办公室取代传统实体办公室并不能帮助公司和他们的员工们变得更有生产力。

    在家工作，也被称为任意地点工作、远程工作或简单的远程办公，通常会带来四个重大变化：

    首先，我们用屏幕监控软件和/或网络摄像头来取代天花板上的办公室安全摄像头，它们会不断地监视我们。Upwork在其成千上万的自由职业者中执行此项操作，要求他们安装一个后台屏幕捕捉实用程序，并确保在整个工作时间内运行。

    其次，我们用Zoom或古老的Skype等会议软件代替会议室，并且学习如何使虚拟会议更具生产力。其主要原则与我们用于传统会议的原则相似：使其有趣。否则，当经理仍在她的窗口中讲话时，我们将切换到Facebook或PokerStars。

    其次，我们用Trello或其他任意的任务跟踪器替换了便利贴和Scrum板。我们让每个人都忙于学习新工具并填充数据。

    第四，也是最重要的是，我们用Slack频道或者HipChat群组取代了办公室的闲聊。每个人不再需要在早上9点到办公室，而是可以通过表情符号展示他们的出勤表现能力。

    作为一名开发者，在远程工作的团队中，你的生活将会有很大的不同。它将充满真正的自由。然而，也会有一些问题：

    证明清白更困难了。你的经理怎么知道你确实在工作，而不是在看电视、和孩子玩耍或者照顾你最喜欢的开源宠物呢？在办公室里，这很容易；你只需坐在显示器前，老板就高兴了。而在家或咖啡店里，你默认是有罪的，除非通过创建新工单、在Slack上发布表情符号和发送带有多个抄送的电子邮件来证明相反情况。

    找到信息更困难了。在办公室里，你总可以大声提问，以便得到最有经验的人的帮助。然而，远程办公时，你将不得不搜索维基页面、谷歌文档，还有，天哪，Git日志。做好准备，几乎没有人会有兴趣分享知识或远程教导你——如果没人看到他在做这些，那还有什么意义呢？你将会独自面对一年前某个被解雇的人写的那份文档。

    更难避免加班。在办公室里，当时间到了，你只需和其他人一起离开。而在远程工作中，尤其是如果团队分布在不同的时区，你将会在周日深夜接到电话和Slack讨论。每个人都期望你全天候都可用，因为你在家工作。而在Skype上将状态改为“请勿打扰”只会冒犯他们，严重损害你的声誉。在家工作环境中，优秀的程序员是那些始终在线的人。

    自我保护更加困难。你不会知道下一次麻烦会从哪里来。在办公室里，你可以直观地观察到领地，嗅到空气中的冲突，看到人们的动作，他们谈论什么，以及下一个受害者是谁。而在远程办公中，你孤立地坐着，从Slack中提取信息碎片。你甚至都可能毫不察觉地成为下一个受害者。简单来说，当你写Java并思考新的单元测试时，他们会在背后捅你一刀。

    不浪费时间更难。听起来有些违反直觉，但在家里的时候，你比在办公室更容易浪费时间。会议将会更频繁地召开，而且由于安排问题、连接质量、软件故障以及缺乏在线沟通技巧等持续问题，这些会议的效果会更差。经理会更多地召开会议，主要是为了让自己相信团队实际在工作；一天之中多次听到你的声音是你忙碌而他是一位好经理的完美证明。

    在更严肃的问题上，我认为当团队的动力仍然基于【时薪】时，远程工作只会使情况变得更糟。远程工作的奴隶仍然是奴隶。而且，这个奴隶更难管理和控制。首先，我们必须转向基于结果的支付模式，然后再转向远程工作。实际上，一旦每个人开始因为他们的成果而获得薪水，而不是他们花费的时间，远程工作将自动发生。他们将简单地不再出现在办公室，但他们的成果将继续为项目做出贡献，而不需要任何网络摄像头或Slack频道。

    毫不奇怪，许多大公司已经决定停止这种远程办公的无稽之谈，让他们的奴隶们重新回到封闭的办公室里；以Google，Yahoo，和Best Buy为例。
