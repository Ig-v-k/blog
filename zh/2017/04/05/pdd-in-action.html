    PDD in Action

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Puzzle Driven development（PDD）是我们团队已经实践了七年以上的一种方法论。使用PDD，我们将任务分解的责任委托给执行者，消除了项目经理的角色。我们一直在使用我们的专有软件来做这个。一个月前，我们将其公开，开源并免费提供。它可以通过0pdd——一个基于GitHub的聊天机器人来使用。

    这是配置它的步骤，分为两步。首先，在 GitHub 中授权给@0pdd（如果你的仓库是私有的）。

    其次，您需要将一个 webhook 添加到您的 GitHub 存储库中：http://www.0pdd.com/hook/github（仅包含 push 事件和任何内容类型）。我实际上建议您通过 ReHTTP 发送 GitHub 通知，并使用以下 URL 进行 webhook：http://p.rehttp.net/http://www.0pdd.com/hook/github。

    现在，你的存储库正在被0pdd监视。每次你执行git push来提交新的更改，它会执行git pull并从GitHub检索你的更改。然后它会运行pdd，一个命令行工具，用于扫描整个代码库（仅限master分支），并查找所有@todo标记的出现。

    对于所有新发现的标记，0pdd 将会向您的代码库的 GitHub 问题跟踪部分提交新问题。

    此外，当您从代码库中删除标记时，0pdd 将立即关闭其创建的问题。

    现在，当问题被分配给程序员时，我们允许他或她采取捷径并将不完整的代码返回到“master”分支。如果代码还没有完成，我们要求程序员在代码中留下”@todo”标记，称之为“puzzles”（谜题）。随后，这些谜题将被分配给其他程序员，依此类推。最终，当大多数谜题得到解决时，问题将被修复。

    0pdd 可以帮助您自动化这个过程，并提供有关整个代码库中所有谜题的当前情况的摘要报告。您甚至可以将一个漂亮的徽章添加到您的 GitHub 仓库中：

    ![PDD状态](http://www.0pdd.com/svg?name=yegor256/0pdd “”)

    如果您点击它，您将看到所有当前存在和以前见过的谜题的完整报告。

    这个机制在很多项目中帮助了我们。你可以免费使用它。这是一个开源的Ruby产品；欢迎贡献。
