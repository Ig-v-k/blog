    ThreeCopies.com—Server-Side Data Backup Service

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我有许多数据资源，这些资源只存在于一个地方，我真的不想失去它们。例如，我有一个托管的PHP网站，以及一个在同一地方托管的MySQL数据库。我还在Amazon DynamoDB拥有一个NoSQL数据库，在Heroku拥有一个PostgreSQL数据库，还有… 嗯，这样的数据库还有很多。如何备份它们一直是我心中的一个问题。

    最直接的方法是租用一个便宜的每月15美元的服务器（或使用现有的服务器），并配置Cron来运行一个自定义的bash脚本，该脚本将从MySQL数据库中提取数据，打包并上传到某个安全的地方，比如亚马逊S3存储桶。然后，我还需要另一个脚本用于PostgreSQL数据库，以及另一个脚本用于FTP文件归档等等。

    这实际上是我多年来一直在做的方式。这种解决方案的缺点总是相同的：

    • 我需要支付服务器费用。

    • 我需要确保服务器始终处于运行状态（Linux远非可靠）。

    • 我也需要备份我的脚本。

    • 每次我想要更改脚本时，我都需要通过SSH登录服务器，记住它们的位置、如何启动等等。

    最大的问题是，每个数据源的所有者都面临着完全相同的一系列问题。多年来，我一直在问自己：“为什么我不能为这些脚本创建一个托管解决方案，来帮助每个人备份他们的数据？”几周前的答案是：“我可以”，于是我创建了ThreeCopies。

    这是一个非常简单的基于托管的Bash脚本执行器，你可以通过网页界面进行编辑。然后我们的一台服务器会启动一个Docker容器（镜像是yegor256/threecopies，Dockerfile在这里）。然后在容器内运行你的脚本。

    脚本每小时、每天和每周启动一次。因此得名为“三份拷贝”。对于数据备份来说，创建具有不同规律的分开拷贝是一个好的实践。此外，您可能希望将不同的数据放入不同的拷贝中。为了帮助您的脚本理解在特定时间运行的是哪个拷贝，我们将$period环境变量传递给它，其值可以是hour、day或week。

    你的脚本如何提取数据、打包数据和存档数据，取决于数据本身。我为大多数常见情况创建了一个简短的备忘单。例如，这是我备份thePMP的MySQL数据库的方法：

    ```bash

    I don’t want to back up every hour

    if [ “${period}” == “hour” ]; then exit 0; fi

    I dump the entire database into the file

    mysqldump –lock-tables=false –host=db.thepmp.com \ –user=thepmp –password=***** \ –databases thepmp > thepmp.sql

    I compress the file

    tgz=”$(date “+%Y-%m-%d-%H-%M”).tgz” tar czf “${tgz}” thepmp.sql

    I upload it to Amazon S3 bucket

    echo “[default]” > ~/.s3cfg echo “accesskey=AKIAICJKH**CVLAFA” » ~/.s3cfg echo “secretkey=yQv3g3ao654Ns***H1xQSfZlTkseA0haG” » ~/.s3cfg s3cmd –no-progress put “${tgz}” “s3://backup.yegor256.com/${tgz}” ```

    脚本的输出可通过网络界面获得，这是这个解决方案的另一个好处。很容易监控出错的情况并重新启动脚本。所有日志都可以通过浏览器获取。不需要SSH，也不需要终端。

    我想说它是 AWS Data Pipeline 的轻量版。 ThreeCopies 做的事情完全一样，但配置更容易，而且更便宜。我每次脚本执行收费0.01美元。而且我实际上是按秒计费，而AWS总是按整个小时计费。以5美元的价格您可以获得500个小时。例如，您在上面看到的脚本大约需要5分钟完成（数据库不大）。如果您跳过每小时执行，就像我上面所做的那样，每个月您将消耗170分钟的服务器时间，这将花费您大约0.34美元每年！我相信这比服务器及其监控要便宜得多。

    在你离开之前还有一件事。ThreeCopies是用Java 8编写的，并且是开源的，在GitHub上找到它（https://github.com/yegor256/threecopies）。请随意检查代码，找到错误，并贡献修复或改进。
