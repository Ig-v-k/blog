    How I Would Re-design equals()

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我想吐槽一下Java的设计，尤其是关于方法Object.equals()和Comparable.compareTo()。多年来，我一直讨厌它们，因为无论我多么努力地尝试喜欢它们，其中的代码看起来都很丑陋。现在我知道了问题所在，以及我如何更好地设计这个”对象对对象比较”的机制。

    假设我们有一个简单的原始类Weight，该类的对象表示某物的重量（以千克为单位）。

    class Weight {
  private int kilos;
  Weight(int k) {
    this.kilos = k;
  }
}

    接下来，我们希望两个重量相同的物体彼此相等。

    new Weight(15).equals(new Weight(15));

    以下是一个这样的方法可能的样子：

    class Weight {
  private int kilos;
  Weight(int k) {
    this.kilos = k;
  }
  public boolean equals(Object obj) {
    if (!(obj instanceof Weight)) {
      return false;
    }
    Weight weight = Weight.class.cast(obj);
    return weight.kilos == this.kilos;
  }
}

    这里的丑陋之处首先是使用instanceof进行类型转换。第二个问题是我们触及了传入对象的内部。这种设计使得Weight的多态行为变得不可能。除了Weight类的实例之外，我们不能将任何其他东西传递给equals()方法。我们不能将其转换为接口并引入多个实现。

    interface Weight {
  boolean equals(Object obj);
}

    这段代码将无法运行：

    class DefaultWeight implements Weight {
  // attribute and ctor skipped
  public boolean equals(Object obj) {
    if (!(obj instanceof Weight)) {
      return false;
    }
    Weight weight = Weight.class.cast(obj);
    return weight.kilos == this.kilos; // error here!
  }
}

    问题在于一个对象决定另一个对象是否相等。这不可避免地导致需要访问私有属性来进行实际比较。

    解决方案是什么？

    这就是我所提供的。无论我们讨论的是什么类型，任何比较都是关于比较两个数字值。我们可以比较重量与重量、文本与文本，或者用户与用户——我们的CPU只能比较数字。因此，我们引入一个新的接口Digitizable：

    interface Digitizable {
  byte[] digits();
}

    接下来，我们介绍一个新的类Comparison，它是两个字节流的比较（我不能确定代码是否完美，我在这里进行了测试，请随意改进并通过拉取请求做出贡献）。

    class Comparison<T extends Digitizable> {
  private T lt;
  private T rt;
  Comparison(T left, T right) {
    this.lt = left;
    this.rt = right;
  }
  int value() {
    final byte[] left = this.lt.digits();
    final byte[] right = this.rt.digits();
    int result = 0;
    int max = Math.max(left.length, right.length);
    for (int idx = max; idx > 0; --idx) {
      byte lft = 0;
      if (idx <= left.length) {
        lft = left[max - idx];
      }
      byte rht = 0;
      if (idx <= right.length) {
        rht = right[max - idx];
      }
      result = lft - rht;
      if (result != 0) {
        break;
      }
    }
    return (int) Math.signum(result);
  }
}

    现在，我们需要Weight来实现Digitizable：

    class Weight implements Digitizable {
  private int kilos;
  Weight(int k) {
    this.kilos = k;
  }
  @Override
  public byte[] digits() {
    return ByteBuffer.allocate(4)
      .putInt(this.kilos).array();
  }
}

    最后，这是我们对它们进行比较的方式：

    int v = new Comparison<Weight>(
  new Weight(400), new Weight(500)
).value();

    这个 v 的值可以是 -1、0 或者 1。在这个特定的情况下，它将是 -1，因为 400 小于 500。

    不再违反封装，不再进行类型转换，在equals()和compareTo()方法中不再有丑陋的代码。Comparison类将适用于所有可能的类型。我们的对象只需实现Digitizable并“提供”其字节以供检查/比较即可。

    这种方法实际上非常接近我之前描述的打印机。
