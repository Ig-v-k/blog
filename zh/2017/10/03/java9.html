    Java是由James Gosling在1995年在Sun Microsystems开发的一种面向对象的语言。对于这种面向对象的主张存在许多争议，我也不确定Java是否比过程性更加面向对象。然而，它被正式定义为面向对象的。

    Java从C/C++继承了许多过程性特征，自其第一个版本起，包括静态方法、NULL、实现继承等。它并不是一种完美的面向对象语言，也不打算成为一种完美的语言，我理解是这样的。关键的想法是创建一种可以“编写一次，到处运行”的语言。然而，这门语言本身也很重要，不仅仅是JVM。它简单而性感。

    Java 5在2004年迈出了一大步，通过添加泛型、for-each循环、可变参数和静态导入等功能来改进语言。然而，引入了注解和枚举，这使得语言从面向对象范式转向了完全不同的过程性范式。

    Java 7在2011年添加了try-with-resource，这是一个很好的举措，符合面向对象范式。

    Java 8在2014年添加了lambda表达式，这是一个很棒的功能，但与面向对象无关。Lambda和Streams API将Java变成了一种混合了面向对象、过程性和函数式范式的语言。还向接口添加了默认方法，将类型转变为代码库。类型变成了代码库！如果你问我，这甚至比实现继承更糟糕。

    现在，Java 9对接口进行了下一步的“改进”，允许它们具有私有方法。类型中的私有静态方法！你能相信吗？下一步会是什么？我猜Java 10会添加属性。
