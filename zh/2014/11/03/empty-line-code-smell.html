    An Empty Line is a Code Smell

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    这个主题听起来像是一个笑话，但实际上并不是。一个空行，在对象方法中作为指令的分隔符使用，是一种代码异味。为什么呢？简单来说，因为一个方法不应该包含“部分”。一个方法应该始终只做一件事，并且它的功能分解应该通过语言结构（例如，新的方法）来完成，而绝不是通过空行。

    看看这个Java类（它有点臭，不是吗？）：

    final class TextFile {
  private final File file;
  TextFile(File src) {
    this.file = src;
  }
  public int grep(Pattern regex) throws IOException {
    Collection<String> lines = new LinkedList<>();
    try (BufferedReader reader =
      new BufferedReader(new FileReader(this.file))) {
      while (true) {
        String line = reader.readLine();
        if (line == null) {
          break;
        }
        lines.add(line);
      }
    }

int total = 0;
for (String line : lines) {
  if (regex.matcher(line).matches()) {
    ++total;
  }
}
return total;
 } } ```

该方法首先加载文件的内容。其次，它计算与提供的正则表达式匹配的行数。那么为什么方法`grep`会有异味呢？因为它执行了两个操作而不是一个---它既加载又进行了grep操作。

如果我们制定一个规则，要避免方法体中的空行，那么为了保持由该空行引入的"关注点分离"，就必须对该方法进行重构。

    final class TextFile { private final File file; TextFile(File src) { this.file = src; } public int grep(Pattern regex) throws IOException { return this.count(this.lines(), regex); } private int count(Iterable lines, Pattern regex) { int total = 0; for (String line : lines) { if (regex.matcher(line).matches()) { ++total; } } return total; } private Iterable lines() throws IOException { Collection lines = new LinkedList<>(); try (BufferedReader reader = new BufferedReader(new FileReader(this.file))) { while (true) { String line = reader.readLine(); if (line == null) { break; } lines.add(line); } return lines; } } }

    
我相信显然这个新类的方法更加内聚且易读。现在每个方法都只做一件事情，很容易理解它在做什么。

这个避免空行的想法也适用于其他语言，不仅仅是Java/C++/Ruby等。例如，这段CSS代码明显需要重构：

```css
.container {
  width: 80%;
  margin-left: auto;
  margin-right: auto;

font-size: 2em; font-weight: bold; } ```

这里的空行在告诉我们（实际上是在对我们大喊大叫），这个`.container`类太复杂了，必须拆分成两个类：

    .wide { width: 80%; margin-left: auto; margin-right: auto; } .important { font-size: 2em; font-weight: bold; } ```

    很不幸地，使用空行来分隔代码块是一种非常普遍的习惯。而且，我经常看到两行甚至三行的空白块，它们都在扮演这种恶意的关注点分隔符角色。

    不言而喻，一个合理设计的类应该只有少数公共方法，而一个合理设计的方法应该有最多十条指令（根据Bob Martin的说法）。方法内的空行会引导我们违反这个很棒的规则，将其变成多页诗歌。

    当然，只需点击“enter”几次并继续在同一个方法中编码，而不是先思考和重构，这样更容易。然而，这种懒惰最终会导致代码几乎无法维护。

    为了防止在您的项目中发生这种情况，请完全停止在方法内部使用空行。理想情况下，应在自动化构建中禁止使用它们。在我们所有的Java项目中使用的静态分析工具qulice.com中，我们创建了一个自定义的Checkstyle检查，禁止在每个方法中使用空行。
