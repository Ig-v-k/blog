    Limit Java Method Execution Time

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    假设您希望允许一个 Java 方法在最多五秒的时间内运行，并且如果超出时间范围，则抛出异常。以下是如何使用 jcabi-aspects 和 AspectJ 来实现：

    public class Resource {
  @Timeable(limit = 5, unit = TimeUnit.SECONDS)
  public String load(URL url) {
    return url.openConnection().getContent();
  }
}

    请记住，你应该在编译后进行类编织，如此处所述。

    让我们讨论一下这个实际的工作原理，但首先，我建议你阅读这篇文章，它解释了AOP方面如何与Java注解一起使用。

    由于@Timeable注解和类编织，对load()方法的每次调用都会被jcabi-aspects中的一个方面拦截。该方面会启动一个新线程，每秒监视方法的执行，并检查它是否仍在运行。

    如果该方法运行超过五秒钟，该线程会对方法的线程调用interrupt()。

    尽管人们普遍期望在此调用时线程应立即终止，但实际上并未发生。这篇文章详细解释了这个机制。让我们简要讨论一下。

    1. interrupt() 在一个线程中设置一个标记；

    2. 线程尽可能频繁地检查interrupted()。

    3. 如果设置了标记，线程将停止并抛出InterruptedException。

    这种方法不会对interrupt()调用做出反应，并且会一直工作，直到JVM被关闭（非常糟糕的设计）。

    public void work() {
  while (true) {
    // do something
  }
}

    这是我们应该如何重构它，以使其对中断请求敏感。

    public void work() {
  while (true) {
    if (Thread.interruped()) {
      throw new InterruptedException();
    }
    // do something
  }
}

    换句话说，只有你的方法自己才能停止。别的什么都不能做到。运行线程不能被另一个线程终止。另一个线程能做的最好的事情就是给你的线程发送一个“消息”（通过interrupt()方法）告诉它该停止了。如果你的线程忽略了这个消息，别人就无能为力了。

    JDK中的大多数I/O操作都是这样设计的。它们在等待I/O资源时会检查线程的中断状态。

    因此，请使用@Timeable注解，但要记住，在某些情况下，线程可能无法被中断。
