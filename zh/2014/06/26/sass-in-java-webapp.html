    SASS in Java Webapp

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    SASS 是一种强大且非常流行的用于编写 CSS 样式表的语言。这就是我在我的 Maven 项目中使用 SASS 的方式。

    首先，我将.css文件的扩展名更改为.scss，并将它们从src/main/webapp/css移动到src/main/scss目录中。

    然后，我配置了sass-maven-plugin（在Maven Central获取其最新版本）。

    <plugin>
  <groupId>nl.geodienstencentrum.maven</groupId>
  <artifactId>sass-maven-plugin</artifactId>
  <executions>
    <execution>
      <id>generate-css</id>
      <phase>generate-resources</phase>
      <goals>
        <goal>update-stylesheets</goal>
      </goals>
      <configuration>
        <sassSourceDirectory>${basedir}/src/main/scss</sassSourceDirectory>
        <destination>${project.build.directory}/css</destination>
      </configuration>
    </execution>
  </executions>
</plugin>

    SASS编译器将会编译src/main/scss中的.scss文件，并将.css文件放置在target/css目录中。

    然后，我配置minify-maven-plugin来压缩/缩小由SASS编译器生成的样式表。

    <plugin>
  <groupId>com.samaxes.maven</groupId>
  <artifactId>minify-maven-plugin</artifactId>
  <configuration>
    <charset>UTF-8</charset>
    <nosuffix>true</nosuffix>
    <webappTargetDir>${project.build.directory}/css-min</webappTargetDir>
  </configuration>
  <executions>
    <execution>
      <id>minify-css</id>
      <goals>
        <goal>minify</goal>
      </goals>
      <configuration>
        <webappSourceDir>${project.build.directory}</webappSourceDir>
        <cssSourceDir>css</cssSourceDir>
        <cssSourceIncludes>
          <include>*.css</include>
        </cssSourceIncludes>
        <skipMerge>true</skipMerge>
      </configuration>
    </execution>
  </executions>
</plugin>

    压缩后的 .css 文件将被放置在 target/css-min 目录中。

    最后一步是配置maven-war-plugin，以便获取.css文件并将其打包到最终的WAR归档文件中。

    <plugin>
  <artifactId>maven-war-plugin</artifactId>
  <configuration>
    [..other configuration options..]
    <webResources combine.children="append">
      <resource>
        <directory>${project.build.directory}/css-min</directory>
      </resource>
    </webResources>
  </configuration>
</plugin>

    That’s it.
