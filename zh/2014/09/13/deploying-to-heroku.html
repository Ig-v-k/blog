    Deploying to Heroku, in One Click

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    已经有一些关于我们使用Rultor来自动化Java和Ruby项目的持续交付周期的文章，包括RubyGems，CloudBees和Maven Central。

    这个描述了如何自动化Heroku部署。当我需要部署一个新版本的Aintshy Web应用程序时，我只需要在GitHub工单中创建一条消息。我只需说 @rultor release 0.1.4，版本0.1.4就会被部署到Heroku上。请参阅GitHub工单#5。

    你可以通过Rultor.com的帮助来做同样的事情，它是一个免费托管的DevOps助手。

    在Heroku.com上创建一个新项目。

    然后安装他们的命令行工具包。

    您应该使用Heroku的命令行工具来验证您的公共SSH密钥。该过程在这里有详细说明，但实际上并不复杂。您只需运行heroku login并输入登录凭据即可。结果是，Heroku将验证您现有的密钥（位于~/.ssh/id_rsa.pub）。

    如果之前没有密钥，它将会自动创建。

    现在，使用rultor remote加密id_rsa和id_rsa.pub（它们位于~/.ssh目录中）。

    $ gem install rultor
$ rultor encrypt -p me/test id_rsa
$ rultor encrypt -p me/test id_rsa.pub

    请使用您的GitHub项目的名称而不是me/test。

    您将获得两个新文件id_rsa.asc和id_rsa.pub.asc。将它们添加到您的项目根目录中，进行提交和推送。这些文件包含您的秘密信息，但只有Rultor服务器才能解密它们。

    在您的项目的根目录中创建一个.rultor.yml文件（参考页面详细解释了此格式）。

    decrypt:
  id_rsa: "repo/id_rsa.asc"
  id_rsa.pub: "repo/id_rsa.pub.asc"
release:
  script: |
    mvn versions:set "-DnewVersion=${tag}"
    git commit -am "${tag}"
    mvn clean install -Pqulice --errors
    git remote add heroku git@heroku.com:aintshy.git
    mkdir ~/.ssh
    mv ../id_rsa ../id_rsa.pub ~/.ssh
    chmod -R 600 ~/.ssh/*
    echo -e \
      "Host *\n  StrictHostKeyChecking no\n  UserKnownHostsFile=/dev/null" \
      > ~/.ssh/config
    git push -f heroku $(git symbolic-ref --short HEAD):master

    您可以将您的文件与 live Rultor configuration of aintshy/hub 进行比较。

    现在是时候看看它是如何工作的了。在GitHub问题跟踪器中创建一个新的票证，并将类似以下内容的内容发布到其中（更多关于Rultor命令的信息，请阅读）：

    @rultor release, tag is `0.1`

    您将在几秒钟内得到回复。其余工作将由Rultor完成。

    Enjoy!

    顺便说一句，如果我解释的某些内容无法正常工作，请随时向Rultor问题跟踪器提交工单。我会尽力帮助您。

    PS。我还建议通过MANIFEST.MF对工件进行版本化，并使用jcabi-manifests在以后读取它们。
