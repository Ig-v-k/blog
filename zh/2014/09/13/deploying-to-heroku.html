    Deploying to Heroku, in One Click

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    已经有一些关于我们使用Rultor自动化Java和Ruby项目的持续交付周期的文章了，包括RubyGems，CloudBees和Maven Central。

    这个描述了如何自动化Heroku部署。当我需要部署一个新版本的Aintshy网站应用时，我只需要在GitHub的工单中创建一条消息。我只需说@rultor release 0.1.4，版本0.1.4就会被部署到Heroku上。请参见GitHub工单#5。

    你可以通过Rultor.com免费托管的DevOps助手来完成相同的操作。

    在Heroku.com上创建一个新项目。

    然后安装他们的命令行工具集。

    你应该使用Heroku的命令行工具来验证你的公共SSH密钥。该过程在这里有说明，但并不复杂。你只需运行heroku login并输入你的登录凭据。作为结果，你的现有密钥（位于~/.ssh/id_rsa.pub）将被Heroku验证。

    如果你之前没有钥匙，它将会自动创建。

    现在，使用 rultor remote 加密 id_rsa 和 id_rsa.pub（它们位于 ~/.ssh 目录中）。

    $ gem install rultor
$ rultor encrypt -p me/test id_rsa
$ rultor encrypt -p me/test id_rsa.pub

    请使用您的 GitHub 项目的名称，而不是 me/test。

    你将得到两个新文件id_rsa.asc和id_rsa.pub.asc。将它们添加到你的项目根目录中，提交并推送。这些文件包含你的秘密信息，但只有Rultor服务器能够解密它们。

    在项目的根目录中创建一个.rultor.yml文件（参考页面详细解释了该格式）。

    decrypt:
  id_rsa: "repo/id_rsa.asc"
  id_rsa.pub: "repo/id_rsa.pub.asc"
release:
  script: |
    mvn versions:set "-DnewVersion=${tag}"
    git commit -am "${tag}"
    mvn clean install -Pqulice --errors
    git remote add heroku git@heroku.com:aintshy.git
    mkdir ~/.ssh
    mv ../id_rsa ../id_rsa.pub ~/.ssh
    chmod -R 600 ~/.ssh/*
    echo -e \
      "Host *\n  StrictHostKeyChecking no\n  UserKnownHostsFile=/dev/null" \
      > ~/.ssh/config
    git push -f heroku $(git symbolic-ref --short HEAD):master

    你可以将你的文件与实时的Rultor aintshy/hub的配置进行比较。

    现在是时候看看它是如何工作的了。在GitHub问题追踪器中创建一个新的工单，并将类似以下内容的内容发布到工单中（了解更多关于Rultor命令的信息）。

    @rultor release, tag is `0.1`

    您将在几秒钟内收到回复。剩下的将由Rultor完成。

    Enjoy!

    顺便一提，如果有任何问题无法按照我所解释的方式运作，请毫不犹豫地在Rultor问题跟踪器上提交一个工单。我会尽力帮助你。

    PS. 我还建议通过 MANIFEST.MF 对工件进行版本控制，并使用 jcabi-manifests 以后读取它们。
