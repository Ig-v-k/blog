    How to Deploy to CloudBees, in One Click

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    当我将一个名为stateful.co的Java Web应用程序部署到CloudBees时，它只需要我30秒的时间，甚至可能更短。最近，我部署了1.6.5版本。你可以在GitHub问题#6中看到所有的过程。

    正如你所看到的，我向Rultor发出了一个命令，它打包、测试并部署了一个新版本到CloudBees。我没有做任何其他事情。

    现在让我们看看你如何做到同样的事情。你如何配置你的项目，以便将其新版本部署到CloudBees只需要几秒钟的时间。

    Since CloudBees is shutting down its PaaS service by the end of December, 2014, this article will have no sense after that.

    Configure the CloudBees Maven Plugin

    将此配置文件添加到您的 pom.xml 中：

    <project>
  [..]
  <profiles>
    <profile>
      <id>cloudbees</id>
      <activation>
        <property><name>bees.appId</name></property>
      </activation>
      <pluginRepositories>
        <pluginRepository>
          <id>cloudbees-public-release</id>
          <url>
            http://repository-cloudbees.forge.cloudbees.com/public-release
          </url>
        </pluginRepository>
      </pluginRepositories>
      <build>
        <pluginManagement>
          <plugins>
            <plugin>
              <artifactId>maven-deploy-plugin</artifactId>
              <configuration>
                  <skip>true</skip>
              </configuration>
            </plugin>
          </plugins>
        </pluginManagement>
        <plugins>
          <plugin>
            <groupId>com.cloudbees</groupId>
            <artifactId>bees-maven-plugin</artifactId>
            <version>1.3.2</version>
            <configuration>
              <appid>${bees.id}</appid>
              <apikey>${bees.key}</apikey>
              <secret>${bees.secret}</secret>
            </configuration>
            <executions>
              <execution>
                <id>deploy-to-production</id>
                <phase>deploy</phase>
                <goals>
                  <goal>deploy</goal>
                </goals>
              </execution>
            </executions>
          </plugin>
        </plugins>
      </build>
    </profile>
  </profiles>
</project>

    这个插件不在Maven中央仓库（遗憾的是）。这就是为什么我们必须指定<pluginRepository>。

    请注意，我们还禁用了 maven-deploy-plugin，因为它会尝试将您的WAR包部署到 <distributionManagement> 部分中的存储库中。我们希望避免这种情况。

    配置文件仅在bees.id属性被定义时激活。这在您的正常开发和测试过程中不会发生，但在Rultor发起的部署周期中会发生，因为我们将在settings.xml中定义此属性（在下面讨论）。

    Secure Access to CloudBees

    在CloudBees中创建一个帐户并注册您的网络应用程序。只要您不需要太多的计算能力，CloudBees是免费的。我认为网络应用程序应该是轻量级的，因此CloudBees的免费层是一个理想的选择。

    创建一个 settings.xml 文件（但不要将其提交到你的代码仓库！）：

    <settings>
  <profiles>
    <profile>
      <id>cloudbees</id>
      <properties>
        <bees.id>stateful/web</bees.id>
        <bees.key></bees.key>
        <bees.secret></bees.secret>
      </properties>
    </profile>
  </profiles>
</settings>

    使用rultor remote对此文件进行加密。

    $ gem install rultor
$ rultor encrypt -p me/test settings.xml

    请使用您的 GitHub 项目的名称，而不是 me/test。

    你应该得到一个 settings.xml.asc 文件；将其添加到你的项目根目录下，然后提交并推送。该文件包含了你的 CloudBees 凭据，但以加密格式保存。除了 Rultor 服务器，没有人能够读取它。

    Configure Versions Plugin

    我建议使用 jcabi-parent。它可以直接配置所需的插件。如果你正在使用它，请跳过此步骤。

    否则，将此插件添加到您的 pom.xml 文件中：

    <project>
  [..]
  <build>
    [..]
    <plugins>
      [..]
      <plugin>
        <groupId>org.codehaus.mojo</groupId>
        <artifactId>versions-maven-plugin</artifactId>
        <version>2.1</version>
        <configuration>
          <generateBackupPoms>false</generateBackupPoms>
        </configuration>
      </plugin>
    </plugins>
  </build>
</project>

    Configure Rultor

    在您的项目的根目录中创建一个.rultor.yml文件（此参考页面详细解释了此格式）。

    decrypt:
  settings.xml: "repo/settings.xml.asc"
release:
  script: |
    mvn versions:set "-DnewVersion=${tag}"
    git commit -am "${tag}"
    mvn clean deploy --settings /home/r/settings.xml

    你可以将你的文件与实时的Rultor stateful.co的配置进行比较。

    Run It!

    现在是时候看看它是如何工作的了。在GitHub问题跟踪器中创建一个新的工单，并将类似的内容发布到其中（详细了解Rultor命令）。

    @rultor release, tag is `0.1`

    你将在几秒钟内收到回复。其余的工作将由Rultor完成。

    Enjoy!

    顺便说一句，如果我的解释有什么问题，不要犹豫，提交一个工单到Rultor问题追踪器。我会尽力帮助你。

    另外，还可以为Heroku（使用jcabi-heroku-maven-plugin）和AWS Elastic Beanstalk（使用jcabi-beanstalk-maven-plugin）执行类似的配置。我可能会分别为它们撰写个别文章。
