    How to Release to Maven Central, in One Click

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    当我将一个新版本的jcabi-aspects发布到Maven中央仓库时，这只需要我花费30秒的时间。甚至可能更少。最近，我发布了0.17.2版本。你可以在GitHub问题#80中看到整个过程。

    正如你所见，我给Rultor发了一个命令，然后它就发布了一个新版本到Maven中央仓库。我没有做任何其他的事情。

    现在让我们看看你如何做到相同的事情。你如何配置你的项目，使得将其新版本发布到Maven中央仓库只需要几秒钟的时间。

    顺便说一下，我假设你正在GitHub上托管你的项目。如果不是的话，整个教程都无法运行。如果你还没有使用GitHub，我强烈建议你迁移到那里。

    Prepare Your POM

    请确保您的pom.xml包含Sonatype所需的所有元素，这些元素在Central Sync Requirements中有解释。我们将部署到Sonatype，他们将会将所有JAR（不仅仅是）工件同步到Maven Central。

    Register a Project With Sonatype

    在Sonatype JIRA创建一个帐户并提出一张工单，请求批准你的groupId。这个OSSRH指南详细解释了这个步骤。

    Create and Distribute a GPG Key

    按照这篇Working with PGP Signatures文章的说明，创建并分发一个GPG密钥。

    当完成此步骤时，您应该有两个文件：pubring.gpg和secring.gpg。

    Create settings.xml

    在上一步创建的两个.gpg文件旁边创建settings.xml文件。

    <settings>
  <profiles>
    <profile>
      <id>foo</id> 
      <properties>
        <gpg.homedir>/home/r</gpg.homedir>
        <gpg.keyname>9A105525</gpg.keyname>
        <gpg.passphrase>my-secret</gpg.passphrase>
      </properties>
    </profile>
  </profiles>
  <servers>
    <server>
      <id>oss.sonatype.org</id>
      <username></username>
      <password></password>
    </server>
  </servers>
</settings>

    在此示例中，9A105525 是您公钥的ID，而 my-secret 是您在生成密钥时使用的密码短语。

    Encrypt Security Assets

    现在，使用rultor远程对这三个文件进行加密。

    $ gem install rultor
$ rultor encrypt -p me/test pubring.gpg
$ rultor encrypt -p me/test secring.gpg
$ rultor encrypt -p me/test settings.xml

    不要使用me/test，而应使用您的GitHub项目的名称。

    你将会得到三个新文件：pubring.gpg.asc、secring.gpg.asc 和 settings.xml.asc。将它们添加到你的项目的根目录中，然后提交并推送。这些文件包含了你的秘密信息，但只有 Rultor 服务器能够解密它们。

    Add Sonatype Repositories

    我建议您使用jcabi-parent作为您项目的父POM。这将使许多后续步骤变得不必要。如果您正在使用jcabi-parent，请跳过此步骤。

    然而，如果您不使用jcabi-parent，则应将这两个存储库添加到您的pom.xml中：

    <project>
  [...]
  <distributionManagement>
    <repository>
      <id>oss.sonatype.org</id>
      <url>https://oss.sonatype.org/service/local/staging/deploy/maven2/</url>
    </repository>
    <snapshotRepository>
      <id>oss.sonatype.org</id>
      <url>https://oss.sonatype.org/content/repositories/snapshots</url>
    </snapshotRepository>
  </distributionManagement>
</project>

    Configure GPG Plugin

    再次，我建议使用jcabi-parent，它会自动配置此插件。如果您正在使用它，请跳过此步骤。

    否则，请将此插件添加到你的 pom.xml 文件中：

    <project>
  [..]
  <build>
    [..]
    <plugins>
      [..]
      <plugin>
        <artifactId>maven-gpg-plugin</artifactId>
        <version>1.5</version>
        <executions>
          <execution>
            <id>sign-artifacts</id>
            <phase>verify</phase>
            <goals>
              <goal>sign</goal>
            </goals>
          </execution>
        </executions>
      </plugin>
    </plugins>
  </build>
</project>

    Configure Versions Plugin

    再次强烈推荐使用http://parent.jcabi.com。它可以直接配置所有所需的插件。如果您正在使用它，请跳过此步骤。

    否则，将此插件添加到您的 pom.xml 文件中：

    <project>
  [..]
  <build>
    [..]
    <plugins>
      [..]
      <plugin>
        <groupId>org.codehaus.mojo</groupId>
        <artifactId>versions-maven-plugin</artifactId>
        <version>2.1</version>
        <configuration>
          <generateBackupPoms>false</generateBackupPoms>
        </configuration>
      </plugin>
    </plugins>
  </build>
</project>

    Configure Sonatype Plugin

    是的，你说得对，http://parent.jcabi.com 在这里也能帮到你。如果你正在使用它，也可以跳过这一步。

    否则，请在您的pom.xml中添加以下四个插件：

    <project>
  [..]
  <build>
    [..]
    <plugins>
      [..]
      <plugin>
        <artifactId>maven-deploy-plugin</artifactId>
        <configuration>
          <skip>true</skip>
        </configuration>
      </plugin>
      <plugin>
        <artifactId>maven-source-plugin</artifactId>
        <executions>
          <execution>
            <id>package-sources</id>
            <goals>
              <goal>jar</goal>
            </goals>
          </execution>
        </executions>
      </plugin>
      <plugin>
        <artifactId>maven-javadoc-plugin</artifactId>
        <executions>
          <execution>
            <id>package-javadoc</id>
            <phase>package</phase>
            <goals>
              <goal>jar</goal>
            </goals>
          </execution>
        </executions>
      </plugin>
      <plugin>
        <groupId>org.sonatype.plugins</groupId>
        <artifactId>nexus-staging-maven-plugin</artifactId>
        <version>1.6</version>
        <extensions>true</extensions>
        <configuration>
          <serverId>oss.sonatype.org</serverId>
          <nexusUrl>https://oss.sonatype.org/</nexusUrl>
          <description>${project.version}</description>
        </configuration>
        <executions>
          <execution>
            <id>deploy-to-sonatype</id>
            <phase>deploy</phase>
            <goals>
              <goal>deploy</goal>
              <goal>release</goal>
            </goals>
          </execution>
        </executions>
      </plugin>
    </plugins>
  </build>
</project>

    Create Rultor Configuration

    在项目的根目录中创建一个 .rultor.yml 文件（参考页面详细解释了这个格式）。

    decrypt:
  settings.xml: "repo/settings.xml.asc"
  pubring.gpg: "repo/pubring.gpg.asc"
  secring.gpg: "repo/secring.gpg.asc"
release:
  script: |
    mvn versions:set "-DnewVersion=${tag}"
    git commit -am "${tag}"
    mvn clean deploy -Pjcabi --settings /home/r/settings.xml

    您可以将文件与 live Rultor configuration of jcabi-aspects 进行比较。

    Run It!

    现在是时候看看它是如何工作的了。在GitHub问题追踪器中创建一个新的工单，并在其中发布类似于以下内容的内容（详细了解Rultor命令）。

    @rultor release, tag is `0.1`

    你将在几秒钟内收到回复。其余的工作将由Rultor完成。

    Enjoy!

    顺便说一句，如果我解释的内容有任何问题，请随时向Rultor问题追踪器提交一个工单。我会尽力帮助你。
