    How to Publish to RubyGems, in One Click

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    当我发布一个新版本的jgd，一个Ruby gem，到RubyGems.org时，只需要我花费30秒的时间。下面是我如何发布了版本1.5.1的bug修复，在GitHub问题 #6中：

    正如您所看到的，我向Rultor发出了一个命令，它发布了一个新版本到RubyGems。我没有做任何其他操作。

    现在让我们看看你可以如何做到相同的事情。你可以如何配置你的项目，以便将其新版本发布到RubyGems.org只需要你几秒钟的时间。

    顺便说一下，我假设你的项目托管在GitHub上。如果不是的话，整个教程将无法使用。如果你还没有在GitHub上，我强烈建议你转移到那里。

    Create RubyGems Account

    在RubyGems.org上创建一个账户。

    Create rubygems.yml

    创建一个 rubygems.yml 文件（你可能已经有了它，名为 ~/.gem/credentials）。

    :rubygems_api_key: d355d8940bb031bfe9acf03ed3da4c0d

    你应该从RubyGems获取这个API密钥。要找到你的API密钥，在登录到RubyGems.org后，点击你的用户名，然后点击“编辑个人资料”。

    Encrypt rubygems.yml

    现在，使用rultor远程加密rubygems.yml：

    $ gem install rultor
$ rultor encrypt -p me/test rubygems.yml

    请使用您的 GitHub 项目的名称，而不是 me/test。

    您将获得一个名为 rubygems.yml.asc 的新文件。将此文件添加到您的项目根目录中，提交并推送。该文件包含您的秘密信息，但只有Rultor服务器可以解密它。

    Prepare Gemspec

    在您的 gemspec 文件中，确保使用 1.0.snapshot 作为版本号。

    
# coding: utf-8

Gem::Specification.new do |s| # ... s.version = '1.0.snapshot' # ... end ```

这个版本名称将在部署过程中被Rultor替换。

## Configure Rultor

在您的项目的根目录中创建一个`.rultor.yml`文件。

    decrypt: rubygems.yml: “repo/rubygems.yml.asc” release: script: | rm -rf *.gem sed -i “s/1.0.snapshot/${tag}/g” foo.gemspec gem build foo.gemspec chmod 0600 /home/r/rubygems.yml gem push *.gem –config-file /home/r/rubygems.yml

    
在这个例子中，用您的宝石的名称替换`foo`。

## Run It!

现在是时候看看它是如何工作的了。在GitHub问题跟踪器中创建一个新的工单，并将类似以下内容的内容发布到其中（了解更多关于[Rultor命令](https://doc.rultor.com/basics.html)的信息）。

    @rultor release, tag is 0.1 ```

    你将在几秒钟内得到一个响应。剩下的事情将由Rultor完成。

    Enjoy!

    顺便说一句，如果我解释的东西不能正常工作，请毫不犹豫地提交一个工单到Rultor问题跟踪器。我会尽力帮助你。
