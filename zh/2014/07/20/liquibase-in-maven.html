    Liquibase with Maven

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Liquibase is a migration management tool for relational databases. It versionalizes schema and data changes in a database; similar to the way Git or SVN works for source code. Thanks to their Maven plugin, Liquibase can be used as a part of a build automation scenario.

    Maven Plugin

    假设您正在使用MySQL（PostgreSQL或任何其他数据库配置将非常相似）。

    在pom.xml中添加liquibase-maven-plugin（在Maven Central获取其最新版本）。

    <project>
  [...]
  <build>
    [...]
    <plugins>
      <plugin>
        <groupId>org.liquibase</groupId>
        <artifactId>liquibase-maven-plugin</artifactId>
        <configuration>
          <changeLogFile>
            ${basedir}/src/main/liquibase/master.xml
          </changeLogFile>
          <driver>com.mysql.jdbc.Driver</driver>
          <url>jdbc:mysql://${mysql.host}:${mysql.port}/${mysql.db}</url>
          <username>${mysql.login}</username>
          <password>${mysql.password}</password>
        </configuration>
      </plugin>
    </plugins>
  </build>
</project>

    要检查它是否正常工作，请运行 mvn liquibase:help。

    我建议您将数据库凭据保存在settings.xml文件和它们各自的配置文件中。例如：

    <settings>
  <profiles>
    <profile>
      <id>production</id>
      <properties>
        <mysql.host>db.example.com</mysql.host>
        <mysql.port>3306</mysql.port>
        <mysql.db>example</mysql.db>
      </properties>
    </profile>
    <profile>
      <id>test</id>
      <properties>
        <mysql.host>test-db.example.com</mysql.host>
        <mysql.port>3306</mysql.port>
        <mysql.db>example-db</mysql.db>
      </properties>
    </profile>
  </profiles>
</settings>

    当你运行Maven时，不要忘记打开其中一个配置文件。例如：mvn -Pproduction。

    Initial Schema

    我假设您已经拥有一个带有架构（表、触发器、视图等）和一些数据的数据库。您应该对其进行“逆向工程”，并为Liquibase创建一个初始的模式文件。换句话说，我们应该告知Liquibase我们目前所处的状态，以便它从这一点开始应用更改。

    Maven插件不支持此功能，因此您必须直接运行Liquibase。但是，这并不难。首先，运行mvn liquibase:help以下载所有工件。然后，用您的实际凭据替换占位符。

    $ java -jar \
  ~/.m2/repository/org/liquibase/liquibase-core/3.1.1/liquibase-core-3.1.1.jar \
  --driver=com.mysql.jdbc.Driver \
  --url=jdbc:mysql://db.example.com:3306/example \
  --username=example --password=example \
  generateChangeLog > src/main/liquibase/2014/000-initial-schema.xml

    Liquibase将分析您当前的数据库模式，并将其自己的模式复制到src/main/liquibase/2014/000-initial-schema.xml中。

    Master Changeset

    现在，创建XML主更改集并将其保存到 src/main/liquibase/master.xml。

    <databaseChangeLog
  xmlns="http://www.liquibase.org/xml/ns/dbchangelog"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://www.liquibase.org/xml/ns/dbchangelog
    http://www.liquibase.org/xml/ns/dbchangelog/dbchangelog-2.0.xsd">
  <includeAll path="src/main/liquibase/2014" />
</databaseChangeLog>

    这是Liquibase的入口点。它从这个文件开始，并加载src/main/liquibase/2014中的所有其他可用的变更集。它们应该是.xml或者.sql。我建议您主要使用XML，因为它更容易维护且速度更快。

    Incremental Changesets

    让我们创建一个简单的变更集，将一个新的列添加到现有的表中：

    <databaseChangeLog xmlns='http://www.liquibase.org/xml/ns/dbchangelog'
  xmlns:xsi='http://www.w3.org/2001/XMLSchema-instance'
  xsi:schemaLocation='http://www.liquibase.org/xml/ns/dbchangelog
    http://www.liquibase.org/xml/ns/dbchangelog/dbchangelog-2.0.xsd'>
  <changeSet id="002" author="Yegor">
    <sql>
      ALTER TABLE user ADD COLUMN address VARCHAR(1024);
    </sql>
  </changeSet>
</databaseChangeLog>

    我们将这个文件保存在src/main/liquibase/2014/002-add-user-address.xml中。在大型项目中，您可以按照它们所属的票据名称命名文件。例如，045-3432.xml 表示来自票号3432的第45个变更集。

    重要的是在文件名前面有这个数字前缀，以便正确排序。我们希望更改按照它们的正确时间顺序应用。

    就这样了。我们准备运行 mvn liquibase:update -Pproduction，我们的生产数据库将会被更新——user 表将会新增一列。
