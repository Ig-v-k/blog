    Basic HTTP Auth for S3 Buckets

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Amazon S3 是一个简单且非常有用的二进制对象（也称为“文件”）存储服务。要使用它，您需要在其中创建一个具有唯一名称的“桶”，并上传您的对象。

    之后，AWS 通过他们的 RESTful API 保证您的对象可供下载。

    几年前，AWS 推出了一个名为 静态网站托管 的 S3 功能。

    通过静态网站托管，您只需打开此功能，您桶中的所有对象都可以通过公共 HTTP 访问。这是一个非常棒的功能，适用于托管静态内容，例如图片、JavaScript 文件、视频和音频内容。

    在使用托管功能时，您需要更改 DNS 中的 CNAME 记录，使其指向 www.example.com.aws.amazon.com。更改 DNS 条目后，您的静态网站将像通常情况下一样可在 www.example.com 上访问。

    然而，在使用 Amazon S3 时，无法保护您的网站，因为内容完全是静态的。这意味着您不能在前端拥有登录页面。使用此服务时，您可以将对象设置为完全公开以供任何人在线查看，或者为对象分配访问权限，但仅限通过 RESTful API 连接的用户。

    然而，我对此服务的使用情况稍微复杂一些。我希望将我的静态内容作为 S3 对象进行托管。然而，我希望通过确保只有少数人可以使用他们的 Web 浏览器访问内容来实现这一点。

    HTTP Basic Authentication

    HTTP 协议提供了一个很好的 “基本访问认证”功能，不需要任何额外的网站页面。

    当一个 HTTP 请求到达服务器时，它不会传送内容，而是回复一个 401 状态响应。这个响应字面上的意思是 “我不知道你是谁，请进行身份验证”。

    浏览器显示其本机登录界面并提示输入用户名和密码。输入登录凭据后，它们会连接在一起，经过 Base64 编码，并添加到下一个请求的 Authorization HTTP 头部中。

    现在，浏览器尝试再次获取相同的网页。但是，这次 HTTP 请求包含一个头部：

    Authorization: Basic am9lOnNlY3JldA==

    以上只是一个例子。在这个例子中，Base64编码部分表示的是joe:secret，其中joe是用户名，secret是用户输入的密码。

    这次服务器有认证信息，并且可以决定该用户是否已认证（他的密码与服务器记录匹配）和是否有权限访问请求的网页。

    s3auth.com

    由于亚马逊不提供该功能，我决定创建一个简单的网络服务s3auth.com，它位于我的亚马逊S3存储桶之前，并实现了HTTP原生的身份验证和授权机制。

    不过，我并不公开我的对象，而是将我的CNAME记录指向relay.s3auth.com。Web浏览器的HTTP请求会先到达我的服务器，然后连接到亚马逊S3，检索我的对象，并将它们作为HTTP响应返回。

    服务器使用根目录下的一个特殊文件.htpasswd来实现身份验证和授权。.htpasswd文件的格式与Apache HTTP Server使用的格式相同，每行表示一个用户。每行包含一个用户的名称和其密码的哈希值。

    Implementation

    我将这个软件开源主要是为了向我的用户保证服务器不会在任何地方存储他们的私人数据，而只作为一个透传服务。因此，这个软件可以在 GitHub 上找到。

    出于隐私和便利性考虑，我只使用 OAuth2 来管理用户账户。这意味着我不知道我的用户是谁。我没有他们的姓名或电子邮件，只有他们在 Facebook、Google Plus 或 GitHub 上的账号编号。当然，我可以通过这些编号找到他们的姓名，但这些信息本来就是公开的。

    服务器是用 Java6 实现的。为了托管它，我使用了一个 Amazon EC2 m1.small 的 Ubuntu 服务器。近来，这个服务器似乎工作正常且稳定。

    Extra Features

    除了身份验证和授权之外，s3auth.com服务器可以渲染页面列表，就像Apache HTTP服务器一样。如果您的存储桶中有一组对象，但是缺少index.html文件，亚马逊S3会提供一个”页面未找到”的结果。相反，我的服务器会显示存储桶中的对象列表，当不存在index.html时，并且可以在一个文件夹内上下导航。

    当您的存储桶启用版本控制功能时，您可以在浏览器中列出任何对象的所有版本。只需在URL末尾添加?all-versions来显示列表。然后，点击一个版本让s3auth.com检索并渲染它。

    Traction

    我创建这个服务主要是为了自己，但显然我不是唯一遇到上述问题的人。目前，s3auth.com 托管了超过300个域名，并每小时传输超过10MB的数据。
