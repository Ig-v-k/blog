    Xembly, an Assembly for XML

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我在几乎所有的项目中都使用XML。尽管有关JSON/YAML的所有争论，但我真诚地认为XML是有史以来最伟大的语言之一。此外，我相信XML的美妙之处在于与相关技术的结合中展现出来。

    例如，您可以以XML格式公开您的数据，并使用XSL样式表将其呈现给最终用户。

    另一个例子是在渲染之前验证相同的数据，以确保结构正确。您可以使用XSD模式来实现这一点。或者，您可以使用XPath查询从整个文档中选择特定的数据元素。

    基本上，这三种技术，XSL、XSD模式和XPath，是使XML变得如此强大的原因。

    然而，XML 有时可能会有不足之处。例如，想象一下您有一个需要稍作修改的现有文档。例如，让我们使用以下内容：

    <accounts>
  [...]
  <acc id='34'>
    <name>Jeffrey</name>
    <balance>305</balance>
  </acc>
  <acc id='35'>
    <name>Walter</name>
    <balance>50090</balance>
  </acc>
  [...]
</accounts>

    上述代码表示一个账户列表。每个账户都有自己的 id 和若干子元素。在我们的示例中，我们需要找到属于 Jeffrey 的账户，并将其余额增加 500。我们应该如何做到这一点？

    嗯，有几个可能的解决方案：

    • 解析SAX文档，更改余额并保存流。

    • DOM解析它，使用XPath找到具体元素，更改值然后打印它；

    • 应用一个带有参数的 XSL 样式表；

    • 应用XQuery小脚本进行更改。

    所有这些方法都有各自的缺点。然而，它们都有一个共同的问题—它们非常冗长。使用上述任何一种方法，您至少需要一页的代码来执行这个相当简单的操作。此外，如果操作的逻辑变得更复杂，所需的代码量会比您预期的增长更快。

    简单来说，XML在文档内部缺乏原始数据操作工具。也许正是这个缺点使得XML在某些人中不受欢迎。

    无论如何，这是我几个月前创建的一个工具：Xembly。它是一种命令式语言，具有几个简单的指令，并且在风格上类似于Assembly。因此，取名为Xembly。使用Xembly，没有循环、条件或变量，只有一系列带有参数的指令。

    让我们创建一个简单的例子。比如说，我们想要将一个新的账号号码36添加到我们的列表文档中。代码看起来会像这样：

    XPATH '/accounts';
ADD 'account';
ATTR 'id', '36';
ADD 'name';
SET 'Donny';
UP;
ADD 'balance';
SET '3400';

    上述内容应该是直观清晰的，但我还是解释一下。首先，XPATH 指令将我们指向由 “/accounts” XPath 查询找到的元素。这将是我们的根元素。在这里我们假设它在文档中存在。因此，如果它不存在，我们的 Xembly 脚本将会因为运行时异常而失败。

    接下来，第2行的ADD指令创建了一个没有子元素或属性的新XML元素。然后，ATTR指令为该元素设置了一个属性。代码接着添加了名为name的新子元素，并使用SET指令将其文本值设置为"Donny"。最后，我们使用UP将指针移回到account元素，添加了名为balance的子元素，并将其值设置为"3400"。

    我们的余额变动任务可以用以下代码在Xembly中表示：

    XPATH '/accounts/account[name="Jeffrey"]/balance';
XSET '. + 500';

    XSET指令设置元素的文本值，类似于SET，但是会使用提供的XPath表达式. + 500进行预先计算。

    Xembly通过DOM执行所有操作。因此，Xembly可以在任何具有内置DOM实现的语言中实现。

    同时，Xembly语言只有一个实现——Java。以下是它的工作原理：

    Iterable<Directive> directives = new Directives()
  .xpath("/accounts")
  .add("account")
  .attr("id", "36")
  .add("name").set("Donny").up()
  .add("balance").set("3400");
new Xembler(directives).apply(document);

    在这个片段中，我使用了一个补充脚本生成器Directives，它可以以流畅的方式生成指令。然后，我使用了Xembler类，它类似于”assembler”，将所有指定的指令应用于org.w3c.dom.Document类的document对象。

    此外，Xembly 可以用于从头开始构建 XML 文档，并作为传统 DOM 构建的替代品。一个快速的例子：

    System.out.println(
  new Xembler(
    new Directives().add("html")
      .add("head")
      .add("title")
      .set("Hello, world!")
  ).xml()
);

    以上的代码片段会产生以下的输出结果：

    <html>
  <head>
    <title>Hello, world!</title>
  </head>
</html>

    对我来说，这似乎更简单和紧凑。
