    PhantomJS as an HTML Validator

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我几个月前创建了phandom，但昨天终于找到时间对它进行了一些必要的修改。所以，现在是解释一下我在一些单元测试中如何使用Phandom的好时机。

    在我开始之前，我应该先谈谈phantomjs，它是一个用于WebKit的JavaScript接口。另一方面，WebKit是一个没有用户界面的网页浏览器。WebKit是一个C++库，它通过DOM调用来实现对HTML内容的操作。例如，下面是一个在example.js中的简单JavaScript代码。

    var page = require('webpage').create();
page.open(
  'http://google.com',
  function() {
    console.log('loaded!');
    phantom.exit(0);
  }
);

    我们使用以下代码从命令行运行phantomjs：

    $ phantomjs example.js

    PhantomJS 创建一个 page 对象（由 phantomjs 内的 webpage 模块提供），然后要求它 open() 一个网页。该对象与 WebKit 进行通信，并将此调用转换为 DOM 指令。之后，页面加载完成。PhantomJS 引擎在第6行终止。

    WebKit使用必要的组件(如CSS、JavaScript、ActionScript等)呈现网页，就像任何标准的Web浏览器一样。

    到目前为止一切顺利，这是使用PhantomJS的传统方式。现在，让我们来给你介绍一下Phandom（代表“PhantomJS DOM”）在Java单元测试中的工作原理。

    为了测试这个，让我们给phantomjs一个HTML页面，并要求它渲染它。当页面准备好后，我们将要求phantomjs展示给我们看这个HTML在WebKit中的样子。如果我们看到了我们需要和期望的元素，那么我们就可以继续了。让我们使用以下示例：

    import com.rexsl.test.XhtmlMatchers;
import org.hamcrest.MatcherAssert;
import org.phandom.Phandom;
public class DocumentTest {
  @Test
  public void rendersValidHtml() {
    Document doc = new Document();
    // This is the method we're testing. It is supposed
    // to return a valid HTML without broken JavaScript
    // and with all required HTML elements.
    String html = doc.html();
    MatcherAssert.assertThat(
      XhtmlMatchers.xhtml(new Phandom(html).dom()),
      XhtmlMatchers.hasXPath("//p[.='Hello, world!']")
    );
  }
}

    当我们使用上述代码时，会发生以下情况。首先，我们从doc对象中获得HTML html作为一个String，然后将其作为参数传递给Phandom。然后，在第13行，我们调用Phandom.dom()方法来获取org.w3c.dom.Document类的一个实例。

    如果我们的HTML包含任何错误的JavaScript代码，dom()方法会产生一个运行时异常并且单元测试会失败。如果HTML干净且WebKit能够正常渲染它，测试就会通过。

    我在几个不同的项目中都使用了这种机制，而且效果相当不错。因此，我强烈推荐它。

    当然，你不应该忘记在你的构建机器上安装必要的phantomjs。为了避免在phantomjs不可用或不存在时导致单元测试失败，我创建了以下补充方法：

    public class DocumentTest {
  @Test
  public void rendersValidHtml() {
    Assume.assumeTrue(Phandom.installed());
    // the rest of the unit test method body...
  }
}
