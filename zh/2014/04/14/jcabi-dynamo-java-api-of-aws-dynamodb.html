    Object-Oriented DynamoDB API

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我是云计算的忠实拥护者，特别是亚马逊网络服务。我真诚地相信在未来几年，大型供应商将托管所有或几乎所有的计算和存储资源。当这种情况发生时，我们将不再太担心停机时间、备份和系统管理员。DynamoDB 是迈向这个未来的一步。

    This looks cool - jcabi-dynamo - a #Java Object layer atop the #DynamoDB SDK - http://t.co/khRFR2joKX #aws

    — Jeff Barr ☁️ (@jeffbarr) September 19, 2013

    DynamoDB是通过RESTful JSON API访问的NoSQL数据库。它的设计相对简单。有表，基本上是数据结构的集合，或者在AWS术语中称为“项”。

    每个项都有一个必需的“哈希”（hash），一个可选的“范围”（range）以及其他一些可选属性。例如，以表depts为例。

    +------+--------+---------------------------+
| dept | worker | Attributes                |
+------+--------+---------------------------+
| 205  | Jeff   | job="manager", sex="male" |
| 205  | Bob    | age=43, city="Chicago"    |
| 398  | Alice  | age=27, job="architect"   |
+------+--------+---------------------------+

    对于Java，亚马逊提供了一个SDK，它将所有的RESTful调用映射为Java方法。这个SDK运行良好，但是以纯过程化风格设计。

    假设我们想要向上面的表格中添加一个新项。RESTful调用putItem的基本形式如下：

    putItem:
  tableName: depts
  item:
    dept: 435
    worker: "William"
    job: "programmer"

    这是亚马逊服务器需要知道的内容，以便在表中创建新项目。以下是你应该通过AWS Java SDK进行此调用的方式：

    PutItemRequest request = new PutItemRequest();
request.setTableName("depts");
Map<String, AttributeValue> attributes = new HashMap<>();
attributes.put("dept", new AttributeValue(435));
attributes.put("worker", new AttributeValue("William"));
attributes.put("job", new AttributeValue("programmer));
request.setItem(attributes);
AmazonDynamoDB aws = // instantiate it with credentials
try {
  aws.putItem(request);
} finally {
  aws.shutdown();
}

    上述脚本运行正常，但有一个主要的缺点——它不是面向对象的。它是命令式过程式编程的一个完美示例。

    为了让您进行比较，让我展示一下我在jcabi-dynamo上所做的。这是我的代码，它以一种面向对象的方式完成了完全相同的事情：

    Region region = // instantiate it with credentials
Table table = region.table("depts");
Item item = table.put(
  new Attributes()
    .with("dept", 435)
    .with("worker", "William")
    .with("job", "programmer")
);

    我的代码不仅更短，而且还使用了封装和分离类的职责。Table类（实际上是由一个类在内部实现的接口）封装了关于表格的信息，而Item封装了项目的详细信息。

    我们可以将一个item作为参数传递给另一个方法，并且所有与DynamoDB相关的实现细节都将对其隐藏。例如，在代码的其他位置：

    void sayHello(Item item) {
  System.out.println("Hello, " + item.get("worker"));
}

    在这个脚本中，我们对DynamoDB或如何处理其RESTful API一无所知。我们仅与 Item 类的一个实例进行交互。

    顺便说一下，jcabi-dynamo 中的所有公共实体都是 Java 接口。由于这个原因，您可以完全测试和模拟该库（但我建议使用 DynamoDB Local 并创建集成测试）。

    让我们来考虑一个更复杂的例子，如果我们使用裸露的 AWS SDK，将需要一页的代码。假设我们想要从我们的表中删除所有担任建筑师工作的工人。

    Region region = // instantiate it with credentials
Iterator<Item> workers = region.table("depts").frame()
  .where("job", Condition.equalTo("architect"));
while (workers.hasNext()) {
  workers.remove();
}

    jcabi-dynamo 在我的几个项目中节省了很多代码行。您可以在rultor-users中看到它的作用。

    该库作为 Maven Central 的 JAR 依赖项提供（从 Maven Central 获取其最新版本）。

    <dependency>
  <groupId>com.jcabi</groupId>
  <artifactId>jcabi-dynamo</artifactId>
</dependency>
