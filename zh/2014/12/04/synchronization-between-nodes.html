    Synchronization Between Nodes

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    当两个或更多的软件模块正在访问同一资源时，它们必须进行同步。这意味着一次只有一个模块应该在处理该资源。如果没有这样的同步，将会发生冲突和冲突。当我们谈论不支持原子事务的”资源”时，这一点尤为重要。

    为了解决这个问题并防止冲突，我们必须在这个情况中引入另一个元素。在访问资源之前，所有的软件模块都应该从一个集中服务器上“捕获”一个锁。一旦对资源的操作完成，模块应该“释放”锁。当一个模块正在捕获锁时，其他模块将无法捕获它。这种方法非常简单且众所周知。然而，我没有找到任何提供这样的锁定和解锁服务的云服务，通过一个RESTful API。所以我决定创建一个——stateful.co。

    这里有一个实际的例子。我有一个Java Web应用，它托管在Heroku上。有三个服务器（也称为“ dynos”）运行相同的.war应用程序。为什么是三个呢？因为Web流量相对活跃，一个服务器的性能不够强大。所以我必须要有三个。它们都运行完全相同的应用程序。

    每个网络应用程序都与Amazon DynamoDB中的一个表一起工作。它会更新表格，将新项目放入其中，有时删除一些项目，并选择它们。到目前为止，一切都很好，但冲突是不可避免的。以下是网络应用程序与DynamoDB之间的典型交互场景的示例（我正在使用jcabi-dynamo）。

    Table table = region.table("posts");
Item item = table.frame()
  .where("name", "Jeff")
  .iterator().next();
String salary = item.get("salary");
item.put("salary", this.recalculate(salary));

    这里的逻辑很明显。首先，我从表posts中检索一项，然后读取其salary，然后根据我的重新计算算法进行修改。问题是，在我重新计算的同时，另一个模块可能开始执行相同的操作。它将从表中读取相同的初始值，然后开始完全相同的重新计算。然后它将保存一个新值，而我也会保存一个新值。最终，我们只会修改Jeff的薪水一次，而用户期望进行两次修改，因为他们使用了两个不同的Web应用程序发起了两个事务。

    在这里的正确方法是首先“锁定”DynamoDB表，甚至在读取薪水之前也要这样做。然后进行修改并最终解锁它。这就是stateful.co如何帮助我。我只需要在stateful.co的Web面板上创建一个新的命名锁，获取我的身份验证密钥，并修改我的Java代码。

    Sttc sttc = new RtSttc(
  new URN("urn:github:526301"), // my GitHub ID
  "9FF3-4320-73FB-EEAC" // my secret key!
);
Locks locks = sttc.locks();
Lock lock = locks.get("posts-table-lock");
Table table = region.table("posts");
Item item = table.frame()
  .where("name", "Jeff")
  .iterator().next();
new Atomic(lock).call(
  new Callable<Void>() {
    @Override
    public void call() {
      String salary = item.get("salary");
      item.put("salary", this.recalculate(salary));
      return null;
    }
  }
);

    如您所见，我将该关键事务封装为Callable，它将在隔离环境中执行。显然，这种方法并不能保证事务的原子性——如果事务的某个部分失败，将不会自动回滚，而DynamoDB表将处于“损坏”状态。

    来自stateful.co的锁定功能能够确保资源使用的隔离性，并且您可以使用任何类型的资源，包括NoSQL表、文件、S3对象、嵌入式软件等等。

    我不应该忘记将这个依赖添加到我的pom.xml文件中。

    <dependency>
  <groupId>co.stateful</groupId>
  <artifactId>java-sdk</artifactId>
</dependency>

    当然，您也可以做同样的事情；该服务完全免费。而且您可以使用任何其他语言，不仅限于Java。顺便说一下，如果您有兴趣，请使用您喜欢的编程语言贡献自己的SDK；我会将其添加到GitHub收集中。
