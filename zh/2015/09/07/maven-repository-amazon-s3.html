    How to Set Up a Private Maven Repository in Amazon S3

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Amazon S3是一个非常适合存放私有Maven构件的地方。我假设你把公共构件放在Maven中央仓库，因为你希望它们对每个人都可用。私有构件是那些你不想让任何人看到，除了你团队的成员。因此，你希望将你的.jar文件部署在那里，并确保它们只对你的团队可见。以下是我们在所有我们的 Java项目中如何实现这一点。

    Create an S3 Bucket

    首先，您需要创建一个新的S3存储桶。我建议您使用项目域名和前缀来命名它。例如，使用repo.teamed.io，repo是前缀，teamed.io是域名。

    对于这个存储桶，无需配置任何权限。只需通过Amazon S3控制台进行创建即可。

    Create an IAM User

    创建一个新的IAM用户。如果您的项目名称是teamed，我建议您将其命名为teamed-maven。

    为用户添加一个新的“内联策略”。

    {
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:*",
      "Resource": [
        "arn:aws:s3:::repo.teamed.io",
        "arn:aws:s3:::repo.teamed.io/*"
      ]
    }
  ]
}

    在这里，repo.teamed.io是您一分钟前创建的S3存储桶的名称。

    确保您有这个新用户的“访问密钥”。它的外观应该类似于这样：

    key: AKIAI9NNNJD5D7X4TUVA
secret: t5tZQCwuaRhmlOXfbGE5aTBMFw34iFyxfCEr32av

    密钥是20个字符（全部大写），而密码是40个字符。

    Extend settings.xml

    将此配置添加到您的~/.m2/settings.xml文件中：

    <settings>
  <servers>
    <server>
      <id>repo.teamed.io</id>
      <username>AKIAI9NNNJD5D7X4TUVA</username>
      <password>t5tZQCwuaRhmlOXfbGE5aTBMFw34iFyxfCEr32av</password>
    </server>
    [...]
  </servers>
  [...]
</settings>

    Configure pom.xml

    请将以下Markdown段落从英文翻译成中文，不要翻译技术术语和专有名词：“将此配置添加到pom.xml文件中：”

    <project>
  <distributionManagement>
    <snapshotRepository>
      <id>repo.teamed.io</id>
      <url>s3://repo.teamed.io/snapshot</url>
    </snapshotRepository>
    <repository>
      <id>repo.teamed.io</id>
      <url>s3://repo.teamed.io/release</url>
    </repository>
  </distributionManagement>
  <repositories>
    <repository>
      <id>repo.teamed.io</id>
      <url>s3://repo.teamed.io/release</url>
    </repository>
  </repositories>
  [...]
</project>

    然后，配置S3 Wagon，同样在pom.xml中：

    <project>
  <build>
    <extensions>
      <extension>
        <groupId>org.kuali.maven.wagons</groupId>
        <artifactId>maven-s3-wagon</artifactId>
        <version>1.2.1</version>
      </extension>
    </extensions>
    [...]
  </build>
</project>

    你已经准备好了。你可以通过在命令行中运行Maven来部署你的构件。

    $ mvn clean deploy

    Configure s3auth.com

    现在，您想在浏览器中以安全模式查看这些文物，需要提供安全凭据。我建议您使用s3auth.com，如《S3存储桶的基本HTTP身份验证》中所解释的那样。

    Configure Rultor

    另一个建议是配置rultor.com，以便自动将您的构件部署到S3。

    首先，使用这个Rultor远程工具加密您的settings.xml文件。

    $ gem install rultor
$ rultor encrypt -p me/test settings.xml

    而不是使用 me/test，你应该使用你的GitHub项目的名称。

    结果，你将得到一个名为 settings.xml.asc 的新文件。将它添加到你的项目根目录中，然后提交并推送。该文件包含你的秘密信息，但只有Rultor服务器才能解密它。

    在你的项目根目录中创建一个名为 .rultor.yml 的文件（Rultor的参考页面详细解释了这个格式）。

    decrypt:
  settings.xml: "repo/settings.xml.asc"
deploy:
  script: |
    mvn clean deploy --settings ../settings.xml

    现在是时候看看它们如何协同工作了。在GitHub问题跟踪器中创建一个新的工单，并将类似以下内容的东西发布到其中（详细了解Rultor命令）。

    @rultor deploy

    您将在几秒钟内收到回复。其余工作将由Rultor完成。

    That’s it.
