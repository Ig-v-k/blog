    XSL Transformation in Java: An Easy Way

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    XSL转换（XSLT）是一种将一个XML文档转换为另一个XML文档的强大机制。然而，在Java中，XML操作相当冗长和复杂。即使对于一个简单的XSL转换，你也需要编写几十行代码，如果需要适当的异常处理和日志记录，可能甚至更多。jcabi-xml是一个小型的开源库，通过几个简单的方法实现了XML解析和XPath遍历，使生活变得更加轻松。让我们看看这个库如何帮助进行XSL转换。

    首先，来看一个实际的例子——rultor.com——一个托管的DevOps助手，可以自动化发布、合并和部署操作。Rultor将与最终用户（也称为”talk”）的每个对话会话保存在DynamoDB记录中。在每个对话中，有多种情况需要处理；所以使用记录的多列并不真正可行。相反，我们只在记录的列中保留每个对话的几个参数（如ID和名称），并将其余的全部放在一个单独的XML列中。

    这大致是我们的DynamoDB表的样子：

    +----+---------------+--------------------------------------+
| id | name          | xml                                  |
+----+---------------+--------------------------------------+
| 12 | jcabi-xml#54  | <?xml version='1.0'?>                |
|    |               | <talk public="true">                 |
|    |               |   <request id="e5f4b3">...</request> |
|    |               | </talk>                              |
+----+---------------+--------------------------------------+
| 13 | jcabi-email#2 | <?xml version='1.0'?>                |
|    |               | <talk public="true">                 |
|    |               |   <daemon id="f787fe">...</daemon>   |
|    |               | </talk>                              |
+----+---------------+--------------------------------------+

    一旦用户在GitHub工单中发布@rultor status，Rultor就必须用一份完整的状态报告来回复关于当前讨论的情况。为了创建这样的文本回答（普通用户不会欣赏XML响应），我们必须从必要的DynamoDB记录中获取那个xml列，并将其转换为简单的英文文本。

    我们通过jcabi-xml和它的类XSLDocument实现了这一点。

    final String xml = // comes from DynamoDB
final XSL xsl = new XSLDocument(
  this.getClass().getResourceAsStream("status.xsl")
);
final String text = xsl.applyTo(xml);

    就是这样。现在让我们看看status.xsl文件中有什么（这只是它的一个框架；完整版本在这里）。

    <xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform"
  version="2.0">
  <xsl:output method="text"/>
  <xsl:template match="/talk">
    <xsl:text>Hi, here is your status report:</xsl:text>
    ...
  </xsl:template>
  <xsl:template match="node()|@*">
    <xsl:copy>
      <xsl:apply-templates select="node()|@*"/>
    </xsl:copy>
  </xsl:template>
</xsl:stylesheet>

    创建XSL文档仅在每个应用程序运行中执行一次是一个好的实践。我们有一个静态实用方法XSLDocument.make()来实现此功能。

    final class Foo {
  private static final XSL STYLESHEET = XSLDocument.make(
    Foo.class.getResourceAsStream("stylesheet.xsl")
  );
  public XML style(final XML xml) {
    return Foo.STYLESHEET.transform(xml);
  }
}

    请注意我们正在使用XSLT 2.0。XSLT的内置Java实现不支持2.0版本，为了使其正常工作，我们使用了这两个Maven Saxon依赖项。

    <dependency>
  <groupId>net.sourceforge.saxon</groupId>
  <artifactId>saxon</artifactId>
  <version>9.1.0.8</version>
  <scope>runtime</scope>
</dependency>
<dependency>
  <groupId>net.sourceforge.saxon</groupId>
  <artifactId>saxon</artifactId>
  <version>9.1.0.8</version>
  <classifier>xpath</classifier>
  <scope>runtime</scope>
</dependency>

    要开始使用 jcabi-xml 进行 XSL 转换，您只需要将以下依赖项添加到您的 pom.xml 文件中即可。

    <dependency>
  <groupId>com.jcabi</groupId>
  <artifactId>jcabi-xml</artifactId>
</dependency>
