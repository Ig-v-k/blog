    Don't Repeat Yourself in Maven POMs; Use Jcabi-Parent

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Maven是一个主要用于Java项目的构建自动化工具。它是一个很好的工具，但它有一个重要的缺点，促使了类似Gradle和SBT的类似工具的创建。这个弱点就是它的配置冗长。Maven从pom.xml文件中获取所有项目构建参数，这是一个可能非常长的XML文件。我见过3000多行的POM文件。考虑到最近的DSL热潮和对XML的恐惧，很多人不喜欢Maven的pom.xml冗长是很合理的。

    但即使你是一个喜欢XML严格性和优雅性的XML粉丝（就像我自己一样），你也不会喜欢在每个项目的pom.xml中重复自己的必要性。如果你在多个项目上工作，代码重复将是巨大的。一个普通的Java Web应用程序使用几十个标准的Maven插件和几乎相同数量的常见依赖项，如JUnit、Apache Commons、Log4J、Mockito等。它们都有自己的版本和配置，如果你想保持项目的稳定性并避免Maven警告，就必须指定它们。因此，一旦插件的新版本发布，你就必须检查你正在工作的项目中的所有pom.xml文件，并在其中进行更新。你显然明白代码重复意味着什么。这是一场灾难。然而，有一个解决方案。

    jcabi-parent是一个非常简单的Maven依赖项，除了一个包含多个预配置依赖项、配置文件和插件的大型pom.xml之外，里面什么都没有。为了在你的项目中重用它们，你只需要将com.jcabi:parent定义为你的父POM。

    ```xml

    4.0.0 com.jcabi parent 0.32.1 [...]

    ```

    这就是你需要的全部内容。现在你可以从pom.xml中删除大部分自定义配置，依赖于jcabi-parent提供的默认配置。它的pom.xml非常庞大且已正确配置。多个项目依赖于它，因此你可以确信你正在使用所有标准插件的最佳配置。
