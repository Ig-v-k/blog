    Temporal Coupling Between Method Calls

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    时间耦合发生在顺序方法调用之间，当它们必须按照特定顺序进行时。这在命令式编程中是不可避免的，但我们可以通过将这些静态过程转变为函数来减少其负面影响。看一个例子吧。

    Here is the code:

    class Foo {
  public List<String> names() {
    List<String> list = new LinkedList();
    Foo.append(list, "Jeff");
    Foo.append(list, "Walter");
    return list;
  }
  private static void append(
    List<String> list, String item) {
    list.add(item.toLowerCase());
  }
}

    你对此有何看法？我相信names()在做什么是很清楚的——创建一个名字列表。为了避免重复，有一个补充的过程叫做append()，它将一个项目转换为小写并添加到列表中。

    这是一个糟糕的设计。

    这是一个过程式设计，方法names()中的行之间存在时间耦合。

    让我先展示一个更好的（虽然不是最好的！）设计，然后我将尝试解释它的好处：

    class Foo {
  public List<String> names() {
    return Foo.with(
      Foo.with(
        new LinkedList(),
        "Jeff"
      ),
      "Walter"
    );
  }
  private static List<String> with(
    List<String> list, String item) {
    list.add(item.toLowerCase());
    return list;
  }
}

    List is not an immutable object. It can be modified after it is created. This means that if the method with() is called multiple times with the same List object, the modifications made in one call will affect the other calls. This violates the principle of immutability. Additionally, creating a new instance of List and populating it with the elements of the original List every time with() is called can be inefficient and slow, especially when dealing with large lists.

    List<String> list = new LinkedList();
Foo.append(list, "Jeff");
Foo.append(list, "Walter");
return list;

    看起来非常干净，是吧？实例化一个列表，向其添加两个项目，然后返回它。是的，现在看起来很干净。因为我们记得append()正在做什么。几个月后，我们会回到这段代码，它会变成这样：

    List<String> list = new LinkedList();
// 10 more lines here
Foo.append(list, "Jeff");
Foo.append(list, "Walter");
// 10 more lines here
return list;

    现在是不是很清楚append()实际上是将"Jeff"添加到list中了？如果我删除那行代码会发生什么呢？会影响最后一行返回的结果吗？我不知道。我需要检查append()方法的具体内容来确认。

    另外，如果先返回list，然后再调用append()呢？这就是可能的”重构”可能对我们的代码所做的操作：

    List<String> list = new LinkedList();
if (/* something */) {
  return list;
}
// 10 more lines here
Foo.append(list, "Walter");
Foo.append(list, "Jeff");
// 10 more lines here
return list;

    首先，我们在list还没有准备好时就过早地返回了。但是有人告诉我这两个append()调用必须在return list之前发生吗？其次，我们改变了append()调用的顺序。同样，有人告诉我按照特定的顺序调用它们很重要吗？

    没有人。任何地方都没有。这被称为时间耦合。

    我们的代码行是耦合在一起的。它们必须按照特定的顺序排列，但是关于这个顺序的知识是隐藏的。很容易破坏顺序，而我们的编译器将无法捕捉到。

    相反，这个设计没有任何”顺序”：

    return Foo.with(
  Foo.with(
    new LinkedList(),
    "Jeff"
  ),
  "Walter"
);

    它只是返回一个列表，这个列表是通过几次调用with()方法构建的。这是一行代码，而不是四行。

    如前所述，在面向对象编程中，理想的方法必须只有一条语句，而这个语句就是return。

    对于验证也是如此。例如，这段代码是有问题的：

    list.add("Jeff");
Foo.checkIfListStillHasSpace(list);
list.add("Walter");

    虽然这个更好：

    list.add("Jeff");
Foo.withEnoughSpace(list).add("Walter");

    See the difference?

    当然，一个理想的方法是使用可组合的装饰器，而不是这些丑陋的静态方法。但如果由于某种原因不可能实现，就不要让这些静态方法看起来像是过程。确保它们始终返回结果，并将这些结果作为进一步调用的参数。
