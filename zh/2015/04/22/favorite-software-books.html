    My Favorite Software Books

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    有很多关于软件工程的书，但只有其中几本是我最喜欢的。我一遍又一遍地读了所有那些我最喜欢的书，如果将来我偶然发现其他不错的书，我可能会更新这篇文章。

    请注意，我试图将最重要的书籍放在列表的顶部。

    对象思维 由大卫·韦斯特所著。这是我读过关于面向对象编程最好的书籍，它完全改变了我对此的理解。我建议你多读几遍。但在阅读之前，试着忘记你过去听到的关于编程的一切。试着从零开始。也许对你也能起作用。

    PMP考试准备，第八版：Rita撰写的通过PMP考试的教材 作者是Rita Mulcahy。这本书是我在项目管理方面的最爱。尽管它关于PMI方法和特别是PMBOK，但对于对管理感兴趣的每个人来说，这都是必读之书。忽略PMBOK的具体细节，重点关注项目管理的哲学和项目经理在其中的角色。

    软件测试的艺术 ，格伦福德·J·迈尔斯等人著。你可以在此处阅读我对这本书的简短评论。这本书完美地解释了测试的哲学，并打破了许多典型的迷思和刻板印象。无论你的职位是什么，如果你在软件行业工作，你都应该了解测试及其基本原则。这是你需要的唯一一本书，以便获得这种理解。

    Growing Object-Oriented Software, Guided by Tests 由 Steve Freeman 和 Nat Pryce 著。这本书包含了你需要了解的关于单元测试的一切。我完全意识到我没有将著名软件工程师 Kent Beck 的著作 包含在这个列表中，因为我一点都不喜欢它。你肯定应该阅读它，只是为了知道发生了什么，但它不会帮助你编写好的测试。相反，阅读这本书，并多次阅读它。

    有效处理旧代码 由Michael Feathers撰写。这本书非常棒，讲述了关于现代软件开发、其陷阱和典型失败的内容。我们现在正在处理的大部分代码都是旧的（也称为开源）。我像读小说一样阅读了这本书。

    持续交付：通过构建、测试和部署自动化实现可靠的软件发布 由Jez Humble和David Farley所著。这是一本关于软件交付、持续集成、测试、打包、版本控制以及其他编程技术的完美书籍。对于任何认真从事软件工程的人来说，这绝对是一本必读之作。

    XML概要（第三版）由Elliotte Rusty Harold和W. Scott Means撰写。XML是我最喜欢的标准，之前我非常讨厌它。在阅读这本书之前，我对所有奇怪的前缀、命名空间、XPath表达式和方案一无所知。但是这本书改变了一切，自从阅读之后，我在任何地方都使用XML。它写得非常好，易于阅读。对于每个人来说都是必备之物。

    Java Concurrency in Practice 由Brian Goetz等人撰写。这是一本非常实用的关于Java多线程的书籍，同时也提供了关于并发性的许多理论知识。我强烈建议你至少阅读一次。

    Effective Modern C++: 42 Specific Ways to Improve Your Use of C++11 and C++14 by Scott Meyers. 无论你使用什么编程语言，这本书都非常有趣且非常有用。它提出了许多关于更好地编写C++代码的重要建议。如果你能理解其中大部分内容，你的Java/Ruby/Python/Scala编程技能将显著提升。

    Code Complete: A Practical Handbook of Software Construction, Second Edition by Steve McConnell。将其视为整洁编码的圣经。多读几遍，并在与同事辩论时将其用作参考手册。它提到了你在现代编程中会看到的最可怕的反模式和最糟糕的实践。要成为一个优秀的程序员，你必须了解它们。

    软件估算：揭秘黑魔法 由 Steve McConnell 撰写。这本书是关于软件工程和其中最棘手的部分——估算的有趣读物。至少阅读一下，以了解这个问题及可能的解决方案。

    编写有效用例 作者：Alistair Cockburn。这是一本古老但非常好的书，实际上你不会在真实项目中使用其中的任何内容，但你会了解到用例的理念，这将引导你正确的思考方向。不要把这本书当作实用的东西；如今几乎没有地方在使用这些用例，但以这种方式确定功能范围的思路是绝对正确的。

    软件需求，第三版 由卡尔·维格斯（作者）和乔伊·贝蒂合著。这是一本关于需求分析的绝佳书籍，也是任何软件项目中第一个且最重要的活动。即使你不是一名分析师，这本书也是必读之作。

    使用 Git 进行版本控制：协同软件开发的强大工具和技术 由 Jon Loeliger 和 Matthew McCullough 所著。这本书是 Git 的实用指南，它是一个版本控制系统。从头到尾阅读它，你将节省很多未来的时间。Git 是版本控制中的事实标准，每个程序员都必须了解其基本原理——不是从备忘单，而是从原始来源。

    JavaScript: The Definitive Guide: Activate Your Web Pages 由David Flanagan著。JavaScript是现代Web的一门语言，这本书对其进行了很好的解释。无论您开发何种类型的软件，都必须了解JavaScript。不要将其视为实用指南（尽管它被称为指南），而应将其视为思考的食粮。对于Java/Ruby/Python开发人员来说，JavaScript提供了很多值得学习的内容。

    CSS：权威指南 由 Eric A. Meyer。CSS 不仅仅涉及颜色和阴影，也不仅仅适用于图形设计师。CSS 是现代 Web 的关键语言。无论您是在 C++ 中使用后端、前端还是桌面应用程序，每个开发人员都必须掌握它。
