    Tacit, a CSS Framework Without Classes

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我在多个项目中使用Bootstrap已经超过两年了，我的不满一直在积累。首先，对于一个小型的Web应用来说，它太庞大了。其次，它并不完全自给自足；无论您如何遵循它的设计原则，最终您还是会得到自己的CSS样式。最重要的是，它的内部设计很混乱。考虑到这一切，我创建了tacit，我的自己的CSS框架，在Hacker News上立刻得到了积极的反馈。

    根据谷歌的解释，Tacit的意思是“不言而喻或暗示的”。这正是framework的概念。它没有一个单独的CSS类，并且可以应用于任何有效的HTML5文档。例如，您有一个HTML文档：

    <!DOCTYPE html>
<html>
  <head>
    <title>Subscribe</title>
  </head>
  <body>
    <section>
      <p>Are you interested in learning more?</p>
      <form>
        <label>Email:</label>
        <input name="email"/>
        <button type="submit">Subscribe</button>
      </form>
    </section>
  </body>
</html>

    这是在Safari中的显示效果：

    现在，我添加了tacit.min.css到它中：

    <!DOCTYPE html>
<html>
  <head>
    <title>Subscribe</title>
    <link rel="stylesheet" type="text/css"
      href="http://yegor256.github.io/tacit/tacit.min.css"/>
  </head>
  <body>
    <section>
      <p>Are you interested in learning more?</p>
      <form>
        <label>Email:</label>
        <input name="email" type="text"/>
        <button type="submit">Subscribe</button>
      </form>
    </section>
  </body>
</html>

    在同款Safari浏览器中看起来是这样的：

    希望你理解了。HTML本身没有任何改变。所有的CSS样式都应用到了标准的HTML元素上。与许多其他CSS框架不同，在Tacit中，你不必在HTML文档中提及CSS类。HTML保持简洁，只以纯HTML5的方式展示数据。

    HTML文档仍然可读可用，但没有好看的图形组件。Tacit以一种非侵入性的方式添加了这个组件。

    当然，在许多项目中，Tacit的默认布局功能可能不足够。在大多数情况下，我还是需要添加自己的CSS类和内联样式。但Tacit为我提供了一个适当的基础。它解决了与响应式表单、表单控件外观、表格、字体和颜色相关的大部分问题。

    Tacit允许我从项目的第一天开始专注于功能。而且功能立即显得很吸引人。我尝试过许多其他框架，包括Bootstrap、Kube和Pure。它们都没有以这个概念为设计目标。它们都将CSS放在HTML前面。在所有这些框架中，CSS是网页设计中最重要的元素，而HTML只是辅助的。

    Tacit采用了不同的方法。在Tacit中，HTML是王者，而CSS只是一个辅助元素，只是让数据看起来更好。
