    JAXB Is Doing It Wrong; Try Xembly

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    JAXB 是一个10年历史的Java技术，它允许我们将一个Java对象转换为XML文档（marshalling），并进行反向操作（unmarshalling）。该技术基于设置器和获取器，在我看来，它违反了面向对象编程的关键原则，将对象转化为被动数据结构。我建议您使用Xembly来代替，将Java对象编组为XML文档。

    以下是JAXB编组的工作原理。假设您有一个需要编组为XML文档的Book类。您需要创建获取器并进行注释：

    import javax.xml.bind.annotation.XmlElement;
import javax.xml.bind.annotation.XmlRootElement;
@XmlRootElement
public class Book {
  private final String isbn;
  private final String title;
  public Book(final String isbn, final String title) {
    this.isbn = isbn;
    this.title = title;
  }
  @XmlElement
  public String getIsbn() {
    return this.isbn;
  }
  @XmlElement
  public String getTitle() {
    return this.title;
  }
}

    然后，您创建一个编组程序，并要求它将Book类的实例转换为XML。

    final Book book = new Book("0132350882", "Clean Code");
final JAXBContext context = JAXBContext.newInstance(Book.class);
final Marshaller marshaller = jaxbContext.createMarshaller();
marshaller.marshal(book, System.out);

    在输出中，您应该期望看到类似这样的内容。

    <?xml version="1.0"?>
<book>
  <isbn>0132350882</isbn>
  <title>Clean Code</title>
</book>

    那么这个有什么问题呢？与对象关系映射存在的问题基本相同，这在《ORM是一种冒犯性的反模式》中有所解释。JAXB将一个对象视为一袋数据，以JAXB希望的方式提取数据并将其转换为XML。对象对此过程没有控制权。因此，对象不再是一个对象，而是一个被动的数据包。

    我们重新设计Book类的理想方法是这样的：

    public class Book {
  private final String isbn;
  private final String title;
  public Book(final String isbn, final String title) {
    this.isbn = isbn;
    this.title = title;
  }
  public String toXML() {
    // create XML document and return
  }
}

    然而，这种方法存在一些问题。首先，存在大量的代码重复。在Java中，构建XML文档是一个相当冗长的过程。如果每个类都必须在其toXML()方法中重新实现它，我们将面临一个大量重复代码的问题。

    第二个问题是我们不知道我们的XML文档应该以何种方式进行包装交付。它可能是一个String，也可能是一个InputStream，或者是org.w3c.dom.Document的一个实例。在每个对象中创建许多toXML()方法肯定会是一场灾难。

    Xembly提供了一个解决方案。正如我之前提到的，它是一种用于XML构建和操作的命令式语言。下面是我们如何借助Xembly实现我们的Book对象的示例：

    import org.xembly.Directive;
public class Book {
  private final String isbn;
  private final String title;
  public Book(final String isbn, final String title) {
    this.isbn = isbn;
    this.title = title;
  }
  public Iterable<Directive> toXembly() {
    return new Directives()
      .add("book")
      .add("isbn").set(this.isbn).up()
      .add("title").set(this.title).up()
      .up();
  }
}

    现在，为了构建一个XML文档，我们应该在对象之外使用这段代码。

    final Book book = new Book("0132350882", "Clean Code");
final String xml = new Xembler(book.toXembly()).xml();

    这个 Xembler 类将把 Xembly 指令转换为 XML 文档。

    这个解决方案的优点是对象的内部不通过 getter 方法暴露，并且对象完全负责 XML 编组过程。此外，这些指令的复杂性可能非常高，远远高于 JAXB 的相当繁琐的注解。

    Xembly 是一个开源项目，欢迎您将您的问题或修正意见提交到 GitHub。
