    JAXB Is Doing It Wrong; Try Xembly

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    JAXB 是一项十年历史的Java技术，允许我们将Java对象转换为XML文档（编组）并反向操作（解组）。该技术基于setters and getters，在我看来，它违反了面向对象编程的关键原则，将对象转化为被动的数据结构。我建议您改用Xembly来将Java对象编组为XML文档。

    这是JAXB marshalling的工作原理。假设你有一个需要被marshalled成XML文档的Book类。你必须创建getter方法并对其进行注解。

    import javax.xml.bind.annotation.XmlElement;
import javax.xml.bind.annotation.XmlRootElement;
@XmlRootElement
public class Book {
  private final String isbn;
  private final String title;
  public Book(final String isbn, final String title) {
    this.isbn = isbn;
    this.title = title;
  }
  @XmlElement
  public String getIsbn() {
    return this.isbn;
  }
  @XmlElement
  public String getTitle() {
    return this.title;
  }
}

    然后您创建一个编组器并要求它将Book类的一个实例转换为XML。

    final Book book = new Book("0132350882", "Clean Code");
final JAXBContext context = JAXBContext.newInstance(Book.class);
final Marshaller marshaller = jaxbContext.createMarshaller();
marshaller.marshal(book, System.out);

    你应该期望输出类似以下这样的内容：

    <?xml version="1.0"?>
<book>
  <isbn>0132350882</isbn>
  <title>Clean Code</title>
</book>

    那它有什么问题呢？基本上与对象关系映射一样有问题，这在《ORM Is an Offensive Anti-Pattern》中有解释。JAXB将对象视为一袋数据，提取数据并按照JAXB的要求将其转换为XML。对象对这个过程没有控制权。因此，对象不再是一个对象，而是一个被动的数据包。

    一个理想的方法是以这种方式重新设计我们的类Book：

    public class Book {
  private final String isbn;
  private final String title;
  public Book(final String isbn, final String title) {
    this.isbn = isbn;
    this.title = title;
  }
  public String toXML() {
    // create XML document and return
  }
}

    然而，这种方法存在一些问题。首先，存在大量的代码重复。在Java中，构建XML文档是一个相当冗长的过程。如果每个类都必须在其toXML()方法中重新实现它，我们将面临重复代码的大问题。

    第二个问题是我们不知道我们的XML文档应该以什么样的方式进行封装传递。它可能是一个String或一个InputStream，或者可能是org.w3c.dom.Document的实例。在每个对象中创建多个toXML()方法肯定是一场灾难。

    Xembly 提供了一个解决方案。正如我之前 提到过 的那样，它是一种用于XML构建和操作的命令式语言。下面是我们如何通过使用Xembly来实现我们的 Book 对象:

    import org.xembly.Directive;
public class Book {
  private final String isbn;
  private final String title;
  public Book(final String isbn, final String title) {
    this.isbn = isbn;
    this.title = title;
  }
  public Iterable<Directive> toXembly() {
    return new Directives()
      .add("book")
      .add("isbn").set(this.isbn).up()
      .add("title").set(this.title).up()
      .up();
  }
}

    现在，为了构建一个XML文档，我们应该在对象之外使用这段代码：

    final Book book = new Book("0132350882", "Clean Code");
final String xml = new Xembler(book.toXembly()).xml();

    这个Xembler类将Xembly指令转换为XML文档。

    这个解决方案的美妙之处在于通过getter方法不会暴露对象的内部，并且对象完全负责XML编组过程。此外，这些指令的复杂性可能非常高—远远高于JAXB中相对繁琐的注解。

    Xembly 是一个开源项目，所以请随时将您的问题或更正提交到 GitHub。
