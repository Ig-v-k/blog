    If. Then. Throw. Else. WTF?

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    这是我始终无法理解的代码：

    if (x < 0) {
  throw new Exception("X can't be negative");
} else {
  System.out.println("X is positive or zero");
}

    我一直在努力寻找一个合适的隐喻来解释它的不正确之处。今天我终于找到了。

    如果-那么-否则是一种程序化编程的分支机制。CPU要么向左走然后做某事，要么向右走然后做其他事情。想象一下自己开车并看到这个标志：

    这看起来很合理，不是吗？如果你不开卡车，你可以选择左车道。否则你应该选择右车道。两条车道在一段时间后会汇合。无论你选择哪条车道，最终都会在同一条路上。这就是这个代码块的作用。

    if (x < 0) {
  System.out.println("X is negative");
} else {
  System.out.println("X is positive or zero");
}

    现在，请试着想象一下这个标志：

    这对我来说看起来非常奇怪，而且你永远不会在任何地方看到这个标志，因为死胡同意味着一个“结束”，一个句号，一个终点。在死胡同标志后面画一个车道的意义是什么？没有意义。

    这是一个正确的代码块的样式：

    if (x < 0) {
  throw new Exception("X can't be negative");
}
System.out.println("X is positive or zero");

    循环的情况也是如此。这样是错误的：

    for (int x : numbers) {
  if (x < 0) {
    continue;
  } else {
    System.out.println("found positive number");
  }
}

    尽管这是正确的:

    for (int x : numbers) {
  if (x < 0) {
    continue;
  }
  System.out.println("found positive number");
}

    死胡同之后没有道路！如果你画出来的话，你的代码看起来就像这个非常有趣的片段，我几年前在审查一家非常严肃的公司某位高薪开发者编写的源代码时发现的。

    if (x < 0) {
  throw new Exception("X is negative");
  System.exit(1);
}

    Don’t do this.
