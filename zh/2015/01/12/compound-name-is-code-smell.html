    A Compound Name Is a Code Smell

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    你命名变量时会使用像 textLength, table_name, 或 current-user-email 这样的命名吗？这三个都是由多个单词组成的复合名称。尽管它们看起来比 name, length, 或 email 更具描述性，但我强烈建议避免使用它们。我认为，比名词更复杂的变量名是一种代码异味。为什么呢？因为当变量的范围变得非常庞大和复杂时，我们通常会给它一个复合名称，简单的名词会听起来模糊不清。而庞大、复杂的范围是一个明显的代码异味。

    亚当和夏娃没有姓氏。在伊甸园中，他们是独一无二的，就像旧约中的许多其他人物一样。后来为了解决歧义问题，才发明了姓氏和中间名。为了保持你的方法和类的简洁和稳固，避免歧义，请尽量给你的变量和方法起一个独特的单词名称，就像亚当和夏娃是由你知道的那个人命名的一样。

    此外，冗余变量也是不好的。
