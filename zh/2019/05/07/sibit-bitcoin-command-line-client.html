    Sibit Demonstrates How Bitcoin Works

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    比特币 对我来说一直是一个巨大的技术谜团。我读过的所有关于它的文章听起来都非常复杂，难以理解。直到我被一个任务困住：我必须将我们实验性的非区块链加密货币Zold与比特币进行整合。我不得不研究比特币的架构，而我找到了这个简短而简单的视频（我强烈推荐你观看）。我成功地实现了整合，并且理解了区块链的工作原理。以下是我的简短总结，希望对你有所帮助。

    首先，有一个私钥，它是一个短小的文本片段，例如：

    c93a36feb31712c390a78b37337cb85d45d3b2f9f6e55108bde32477cbabac5f

    我是如何生成它的？它是随机的。你也可以生成一个。安装Sibit，我的Ruby宝石，并运行以下命令（它是一个命令行工具）：

    $ sibit generate
c93a36feb31712c390a78b37337cb85d45d3b2f9f6e55108bde32477cbabac5f

    每次调用sibit generate，你都会得到一个新的私钥。它只是一段随机的文本（或者说是一个大的十六进制数）。然后，你可以根据你的私钥创建一个地址。每个私钥都对应一个地址。下面是创建地址的方法：

    $ sibit create c93a36feb31712c390a78b37337cb85d45d3b2f9f6e55108bde32477cbabac5f
1K3JgsdRbbUDUgTGo4gvUXUzRZjBse1TYe

    你把它给想给你付款的人。比特币交易不是你想象中的银行交易。比特币交易有一些“输入”和“输出”地址。比如，三个输入地址每个有10个比特币，两个输出地址分别有25个比特币和5个比特币。所有输入的总和必须等于所有输出的总和。

    因此，在一次交易中，你可以在地址之间转移大量的数字货币。你可以将一百个输入地址发送到另外一百个输出地址。当我发现这一点时，我感到有些惊讶。交易可能会非常庞大。并且大小很重要。它们以字节计算。一笔只有一个输入和一个输出的交易将消耗大约220字节。请记住这个数字，我们很快会回到它。

    正如你已经可以想象的那样，为了计算你的地址上有多少钱，你只需查看整个数据库中的所有交易（已经有超过4亿）并查看有多少交易将你的地址作为它们的输出（你的输入）。然后，扣除任何你的输出和他人的输入的交易。你就得到了余额。有很多公共和免费的网络服务可以为你完成这项工作。你不需要自己搜索整个数据库，只需使用Blockchain API（例如），他们将为你提供有关任何比特币地址的信息。例如，试试这个地址：

    $ sibit balance 1MUhYhaBqzgpwL1focqJNhTymdXbyky9UY
20000000

    它是以satoshi的形式存储的，相当于0.2 BTC。点击这里，您将看到该地址的输入和输出。只有一笔交易提到了这个地址。该交易只有一个输入和两个输出。其中一个输出是我的。我得到了0.2个比特币，它们仍然在这里，因为没有将它们移动到其他地方的交易。

    简而言之，区块链数据库是一个包含每个交易的输入和输出的大型交易列表。一旦新的交易进入该数据库，几个地址的余额就会发生变化。数据库只会增长，之前的交易不可删除或修改。因此，一旦您的交易进入，资金就会从一个地址转移到另一个地址。

    要创建一个交易，您需要知道所有的输入地址、它们的私钥和所有的输出地址。如果您要发送的金额大于您在单个地址中的余额，您需要多个输入。比特币中有一个被称为钱包的东西，它是一种软件，如Electrum，可以跟踪您所有的地址，并知道如何创建一个交易，结合多个地址并将它们用作输入。就是这样。

    所以，您知道输入，您指定输出，并且您应该保留一小笔未使用的金额。假设您的输入有10,000 satoshi，并且您指定了一个8,000 satoshi的输出。您将剩余的2,000 satoshi未使用。它们被称为矿工费。您将交易发送到所有比特币节点，它们将其放入其内存池，并试图将几千个交易组合成一个区块。它们一起尝试，只有一个成功，大约每十分钟一次。一旦获胜者成功地解决一个区块，它就会赢得您交易中未使用的那2,000 satoshi。区块中的每笔交易都向获胜者支付一些小费。

    有可能不支付任何费用吗？是的。然而，您在交易中留下的费用越小，您的交易进入下一个区块的机会就越低。每个矿工都希望在每个区块中获得最大利益，并在尝试挖矿时将最“慷慨”的交易放入区块中。

    费用取决于交易的字节大小。由于任何交易可能包含许多输入和输出，所以只有大小才有影响。例如，我在这笔交易中支付的费用是8,971 satoshi，约为每字节40 satoshi，因为我的交易包含225个字节。

    要将您的交易发送到所有比特币节点，有两种选择。第一种方法是启动自己的完整节点，它将通过比特币协议与其他节点通信。第二种选择是使用HTTP中继之一，在其中通过POST HTTP请求发布您的交易，中继将其发送到所有比特币节点；这是其中之一。

    Sibit可以自动化此过程。您只需说：

    $ sibit pay 200000 L \
  1K3J...se1TYe:c93a36feb3...e55108bde32477cbabac5f \
  153dF1xKyVX5X8brBDroaqKkPmcB8kLtDB \
  1K3JgsdRbbUDUgTGo4gvUXUzRZjBse1TYe

    在这里，第一个参数是您要发送的金额。第二个参数是您想要保持不变的手续费。我为您做得更简单，以帮助您避免计算。只需说“S”、“M”、“L”或“XL”，手续费将被计算得更或多或少正确。第三个参数是地址和它们的私钥的列表，用冒号分隔。第四个参数是目标地址（Sibit使得只能发送到一个地址）。最后一个参数是找零将被发送到的地址。

    如果您的输入中有50,000 satoshi，并且您想要向朋友发送15,000 satoshi，您必须找到一个地方将剩下的35,000（减去矿工费）发送过去。您只需要提供您自己的地址，找零将会送到那里。

    这基本上是您使用比特币和区块链所需了解的全部内容。我漏掉了什么吗？
