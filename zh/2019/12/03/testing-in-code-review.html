    Does Code Review Involve Testing?

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    当你审查来自他人的拉取/合并请求时，你会检出分支并运行构建吗？我通常不会，但是有些人会。他们显而易见的理由是：运行构建，甚至手动测试产品，可以帮助发现更重要的错误。仅仅查看源代码可能无法揭示最近引入的HTML/CSS的所有视觉缺陷，例如。最好的做法是检出分支，启动Apache，在Chrome中打开网站，并查看存在什么问题。然后，截图，将其附加到拉取请求中，并将其退还给作者。但是我不同意这一点，以下是原因。

    这个讨论并不新鲜，参考这个和这个在SO上的帖子。然而，似乎那些回答都忽略了关键点。

    众所周知，在任何软件项目中都存在两个相互冲突的角色：构建者和销毁者，也被称为程序员和测试人员。程序员添加新功能并修复错误。他们的成果是所创建功能的数量：数量越多越好。而测试人员则是在破坏产品并报告错误，竭尽所能证明产品还不准备好交付给客户。在某个时间点，团队（或管理层）决定战斗结束，产品可以交付。

    由于这种必要的冲突，质量得以实现。

    程序员在代码通过合并流水线时完成了他们的任务：更改是在他们的笔记本电脑上进行的，单元测试在本地通过，静态分析没有报错，构建是干净的，分支被合并到主干。这是程序员停下来并获得奖金的地方。

    测试人员完成了他们的任务，当他们成功在部署到暂存或生产环境的产品中发现一个新的缺陷时：已经找到、报告并且项目方已经接受了这个 bug。在这里，测试人员停止工作并获得他们的奖金。

    这是明显的。如果不是，请阅读 Glenford Myers 的《软件测试艺术》或 Yegor Bugayenko 的《Code Ahead》。您也可以观看此视频。

    现在，在这个冲突中，代码审查员的位置在哪里？

    我相信代码审查是合并流水线的一部分，与单元测试、静态分析器、代码检查工具、覆盖率控制工具、变异测试工具以及项目中可能希望加入的其他工具一起，目的是使程序员的生活更困难，提高源代码的质量。合并流水线的目标是保护代码库免受程序员的影响。

    代码审查者与代码检查工具或静态分析工具没有什么不同：它们都会阻止有问题的代码分支进入主干。当它们阻止时，它们会给出原因。与代码检查工具类似，他们的工作完成时是在分支中找到问题并解释它。当代码检查工具拒绝一个拉取请求时，它会准确地指出问题所在，包括哪一行以及如何修复。这就是一个好的代码审查者必须做的：找到有问题的行，解释问题，并提出解决方案。

    这就是代码评审员必须为之付费的内容：完成的评审。

    什么是已完成的审查？“一切都好”听起来像是已完成的审查吗？对于一个代码检查工具来说，是的；对于一个代码审查者来说，不是。这个更好：“我发现了三个问题，解释了它们，然后进行了辩论或修复。”代码审查者的工作描述可能是这样的：找到三个最关键的问题，解释它们，并确保它们要么被修复，要么正确地进行了论证。

    评审人员如何发现这三个问题，这取决于他们自己。他们可以通过对代码进行目视检查或运行构建来发现问题。然而，只要他们成功发现问题并确保代码的作者理解这些问题并修复它们或解释为什么无法修复（或者不是问题），他们仍将得到报酬。对代码进行目视检查很快，而检查分支和运行构建则需要更多时间。此外，运行本地构建后发现的错误很难以代码审查的形式来解释。与分支的作者进行讨论将需要更长时间，这意味着完成代码审查需要更多时间，也意味着代码评审人员的效率较低。

    我的观点是，聪明的代码审查者不会这样做，因为这样效率不高。请注意，这并不是因为他们不在乎，而是因为他们知道更好的为项目做贡献的方法。让我们面对现实吧，当面前的分支通过了所有自动检查，如代码风格检查和单元测试，但仍然存在我们只能通过执行代码才能复现的一些错误时，说明我们的自动化测试有问题。一个有效、负责任且贪婪的代码审查者不会告诉代码的作者出了什么问题。相反，会创建一个新的错误，责怪合并流程太弱。当然，这个新的错误会得到奖励。

    因此，作为代码审查人员，您可以在本地使用分支进行工作，对其进行测试并将发现报告给作者。但这对您个人利益来说是不利的，也对项目没有益处。相反，您应该向项目投诉自动化测试的质量低，并将审查暂停。当投诉得到解决时，测试变得更强大，您再回到审查，这次不是由您而是由合并流程拒绝。

    在这种情况下，每个人都是赢家：流程变得更强大，你因为报告的漏洞获得额外奖金，而且审查被拒绝，原因非常明确且可以再现。
