    How to Deploy Maven Artifacts to CloudRepo via Rultor

    The following text is a partial translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    在我的上一篇文章中，我描述了如何在Amazon S3中设置一个私有的Maven存储库，并通过Rultor部署。如果您熟悉管理Amazon Web Services（AWS），S3和AWS Identity and Access Management (IAM)，这是一个很好的解决方案。然而，如果您不熟悉管理AWS账户和相关权限，您可能希望将您的Apache Maven Artifacts存储在一些基于云的存储库管理器中。这里是如何让Rultor将您的Maven依赖项部署到CloudRepo的方法。我与他们的创始人Chris Shellenbarger一起撰写了这篇博文。

    仓库管理器和S3都可以让您的构建工具将软件库存储和检索到远程仓库或存储桶。然而，仓库管理器会处理您需要自己管理的大量工作，而S3解决方案则需要您自行处理。

    虽然AWS非常强大，可以配置成实现仓库管理所需的所有功能，但选择完全托管的解决方案时，有些用例可以直接开箱即用，例如：用户/组管理、Maven特定视图、Webhooks、通知、访问和审核日志、完全托管的安全性等等。简而言之，它们不仅仅是JAR文件的存储，而是云端的Maven仓库。

    一个完全托管的云托管解决方案将需要您支付每月的订阅费用。截至撰写此文时，他们每月收费9美元起。对于开源项目来说是免费的，但如果您可以使用Maven Central，我不知道为什么您还需要它们。好消息是，他们为我的读者提供了特别折扣：前90天免费，而标准为14天。这实际上相当于您首年享受25%的折扣。只需在创建帐户后给他们发送一封电子邮件，他们会为您处理（别忘了提到我的名字，以获得折扣）。

    假设您已经在CloudRepo上创建了一个帐户并设置了用户和Maven仓库，在客户端上部署到CloudRepo需要两个步骤：1）使用凭据配置settings.xml文件，以及2）将您的仓库添加到pom.xml文件的<distributionManagement>部分。

    settings.xml 文件的默认位置是在你的 ~/.m2 目录中。编辑此文件并确保你拥有如下所示的 <server> 条目：

    <settings>
  <servers>
    <server>
      <id>io.cloudrepo</id>
      <username>yegor256@gmail.com</username>
      <password>my-secret</password>
    </server>
  </servers>
</settings>

    通过声明一个server并指定一个id，您可以在您的Maven POM文件中引用该id。当Maven尝试对服务器进行身份验证时，它将在settings.xml文件中查找相应的密钥。

    如果您担心在文件系统中以明文形式存储密码，请查看Maven如何建议修复此问题的链接。

    现在您的凭证已设置好，您必须将您的 pom.xml 指向您的 CloudRepo 存储库。在您的 pom.xml 的 <distributionManagement>（用于上传）部分和 <repositories>（用于下载）部分中添加一个新的 <repository> 元素。

    <project>
  [...]
  <repositories>
    <repository>
      <id>io.cloudrepo</id>
      <url>https://[your-org-name].mycloudrepo.io/repositories/[your-repository-name]</url>
    </repository>
  </repositories>
  [...]
  <distributionManagement>
    <repository>
      <id>io.cloudrepo</id>
      <name>CloudRepo Maven Repository</name>
      <url>https://[your-org-name].mycloudrepo.io/repositories/[your-repository-name]</url>
    </repository>
  </distributionManagement>
</project>

    一旦您在本地环境中进行了构建，您需要使用Rultor将其部署到生产环境中。首先，您需要将凭据存储在Git中，以便Rultor可以在构建和部署时访问它们。然而，非常重要的是永远不要以明文形式将凭据存储在版本控制中（将您的GitHub项目名替换为my/project）。

    $ gem install rultor
$ rultor encrypt -p my/project settings.xml

    这将创建一个名为 settings.xml.asc 的加密版本的设置文件。将此文件添加到项目的根目录中，然后进行提交和推送。它是安全的。只有 Rultor 拥有解密此文件的密钥，因此即使您的源代码暴露给他人，您的凭据也将得到保护。

    要启用Rultor，请在项目的根目录中添加一个.rultor.yml文件，文件内容如下：

    decrypt:
  settings.xml: "repo/settings.xml.asc"
deploy:
  script: |
    mvn clean deploy --settings ../settings.xml

    有关 .rultor.yml 文件的更多信息，请查看Rultor 参考页面。

    现在一切都已经配置完成，通过执行@rultor deploy命令，您应该能够使用Rultor将应用部署到CloudRepo。等待响应，Rultor将处理其余的所有事务。

    That’s it!
