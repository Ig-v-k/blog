    How I Would Re-design equals()

    The following text is a translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我想稍微抱怨一下Java的设计，特别是关于Object.equals()和Comparable.compareTo()这两个方法。多年来，我一直很讨厌它们，因为无论我多么努力想喜欢它们，其中的代码看起来都很丑陋。现在我知道了具体的问题在哪里，以及我会如何更好地设计这个”对象对比”机制。

    假设我们有一个简单的原始类Weight，它的对象表示某物的重量（以千克计）。

    Java类Weight { private int kilos; Weight(int k) { this.kilos = k; } }

    接下来，我们希望两个重量相同的物体能够相互等于。

    Java新的Weight(15).equals(new Weight(15));

    这是这种方法可能的样子：

    java类重量{ private int 公斤; 重量（int 公斤）{ this.公斤 = 公斤; } public boolean equals（Object obj）{ if（！（obj instanceof 重量））{ return false; } 重量重量=重量.class.cast（obj）; return 重量.公斤== this.公斤; } }

    这里的丑陋之处首先是使用instanceof进行类型转换。第二个问题是我们接触到了传入对象的内部。这种设计使得Weight的多态行为变得不可能。除了Weight类的实例，我们无法将任何其他东西传递给equals()方法。我们无法将其转换为接口并引入多个实现。

    Java接口Weight { boolean equals(Object obj); }

    This code will not work:

    java类DefaultWeight实现Weight接口 { // 属性和构造函数省略 public boolean equals(Object obj) { if (!(obj instanceof Weight)) { return false; } Weight weight = Weight.class.cast(obj); return weight.kilos == this.kilos; // 这里有错误！ } }

    问题在于一个对象决定另一个对象是否相等。这必然导致需要访问私有属性以进行实际比较。

    What is the solution?

    这就是我所提供的。无论我们讨论的是什么类型，任何比较都是关于比较两个数字值。无论是比较重量与重量，文本与文本，还是用户与用户，我们的CPU只能比较数字。因此，我们引入一个新的接口”可数字化”：

    Java接口Digitizable { byte[] digits(); }

    接下来，我们介绍一个名为Comparison的新类，它用于比较两个字节流（我不确定代码是否完美，我在这里进行了测试，请随意改进并通过pull request贡献代码）。

    java类Comparison { private T lt; private T rt; Comparison(T left, T right) { this.lt = left; this.rt = right; } int value() { final byte[] left = this.lt.digits(); final byte[] right = this.rt.digits(); int result = 0; int max = Math.max(left.length, right.length); for (int idx = max; idx > 0; --idx) { byte lft = 0; if (idx <= left.length) { lft = left[max - idx]; } byte rht = 0; if (idx <= right.length) { rht = right[max - idx]; } result = lft - rht; if (result != 0) { break; } } return (int) Math.signum(result); } }

    现在，我们需要Weight来实现Digitizable。

    java类Weight实现Digitizable接口{ private int kilos; Weight(int k) { this.kilos = k; } @Override public byte[] digits() { return ByteBuffer.allocate(4) .putInt(this.kilos).array(); } }

    最后，这是我们如何进行比较：

    java int v = new Comparison( new Weight(400), new Weight(500) ).value();

    请将上述代码段从英文翻译成中文，不要更改专有名词：

    这个v的值可以是-1、0或1。在这特定的情况下，它将是-1，因为400小于500。

    不再违反封装，不再进行类型转换，不再在equals()和compareTo()方法中存在丑陋的代码。Comparison类将适用于所有可能的类型。我们的所有对象只需要实现Digitizable接口并“提供”其字节以供检查/比较即可。
