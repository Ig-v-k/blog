    Software Quality Award, 2017

    The following text is a translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    这是软件质量奖的第三年。奖金仍然是相同的——4096美元。规则仍然是相同的。继续阅读。往年的奖项在这里：2015年，2016年。

    The rules:

    一个人只能提交一个项目。

    投稿截止日期为2017年9月1日。

    我会检查提交历史，以确保你是该项目的主要贡献者。

    我保留拒绝任何提交而无需解释的权利。

    所有提交内容都将在此页面上发布（包括被拒绝的内容）。

    结果将于2017年10月15日在本页面和通过电子邮件公布。

    最佳项目将获得4096美元。

    最终决策将由我做出，并且不可更改（尽管我可能会邀请其他人帮助我做出正确的决策）。

    在过去几年中获得任何现金奖励的获奖者不能再次提交申请。

    Each project must be:

    Open source (in GitHub).

    At least 10,000 lines of code.

    At least one year old.

    面向对象（这是我唯一理解的东西）。

    使用这些标准来选择最佳项目。

    What doesn’t matter:

    流行度。即使没有人使用你的产品，它仍然有资格获得这个奖项。我不在乎流行度；质量才是关键。

    编程语言。我相信任何语言，只要使用得当，都可以用来设计一个高质量的产品。

    嗡嗡声和趋势。即使你的项目只是另一个命令行参数解析器，它仍然有资格获奖。我不在乎你的营销职位；质量才是最重要的。

    顺便说一下，如果你想赞助这个奖项并增加奖金，请给我发送电子邮件。

    [21 Oct 2017] My short list includes these six projects (in random order): php-ai/php-ml, vavr-io/vavr, zetaops/ulakbus, mafagafogigante/dungeon, ribtoks/xpiks, javascript-obfuscator/javascript-obfuscator. Tomorrow (hopefully) I will decide how to split $4096.

    [23 Oct 2017] These are my own observations per project from the short list. I will only mention negative things, since all projects are pretty good, no need to say how good they are. I listed problems in order of importance (most critical on top).

    php-ai/php-ml（9.8K行PHP代码，29K行注释）：

    How do you release it?

    获取器、设置器和可变性存在于许多地方。

    NULL在很多地方出现（再次强调，在PHP中不存在方法重载）。

    -ER: Estimator, Classifier, Clusterer, Optimizer, etc.

    构造函数中的代码（是的，我明白这是PHP）。

    empty lines in method bodies

    Score: 5

    vavr-io/vavr（70K行Java代码，834K行代码）：

    How do you release it?

    有一些非常大的“classes”，它们在io.vavr.collection包中非常庞大。

    接口Seq有120多个方法！发生了什么事情？

    Utility classes, static methods

    为什么有些 .java 文件包含多个 Java 类？

    Could not build master: #2147

    Score: 4

    zetaops/ulakbus（25K行的Python代码，707K行的代码总量）：

    How do you release it?

    没有CI，没有测试覆盖率，没有静态分析自动化？

    请看评论者的评论

    Score: 2

    mafagafogigante/地牢（14K行Java代码，88K行代码）：

    发布自动化，但仅针对一个人。

    静态方法，获取方法，设置方法，可变性。

    提交不会链接到问题和PR。

    在方法体内的注释存在于很多地方，这是一种不良的做法。

    Score: 5

    ribtoks/xpiks（180K+行的C/C++代码，739K行的高级代码）：

    No coverage, no static analysis

    类型相当大，具有许多方法。

    Util classes, helpers

    -ERs: CommandManager, SpellCheckWorker, etc.

    我在代码中并没有找到太多的文档。

    提交与问题/PR没有关联。

    Score: 4

    javascript-obfuscator/javascript-obfuscator（72K行JS / TS，400K HoC）：

    Utils and just global functions

    注解驱动的可注入依赖

    -ERs: reader, sanitizer, emitter

    Public attributes in classes

    我相信这里有许多“对象”只是数据传输对象（DTOs）。

    接口以I为前缀，这是一种反模式。

    Score: 4

    今年的总体印象是我收到的垃圾要少得多。提交的项目较少，但质量比前两年要高得多。看到这种趋势我感到很高兴。这对我来说意味着我正在做正确的事情。

    这一次，我更加注重面向对象编程的优雅性和代码库的可维护性。可维护性的关键因素有：

    Automated releases

    Automated static analysis

    Automated builds (CI)

    Automated tests

    纪律良好的承诺，通过问题和PRs

    对于面向对象编程(OOP)的优雅性，我通常会关注反模式的缺失，包括NULL、getters、setters、static、mutability等等。

    今年有两个获胜者：php-ai/php-ml和mafagafogigante/dungeon。但我并不是特别喜欢我在这些代码库中找到的代码。它显然比其他任何人都要好，但并不完美。

    这就是为什么，这是我的决定：我将给每位获胜者仅仅1024美元，而不是2048美元。

    恭喜@itcraftsmanpl赢得php-ml（$1,024），以及恭喜@mafagafogigante赢得dungeon（$1,024）。
