    How We Run as a Non-Root Inside Docker Container

    The following text is a translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    Docker 在其容器中以“root”用户启动一个进程。然而，在某些情况下，这并不方便。例如，PostgreSQL 的 initdb 不喜欢以 root 用户启动，会导致失败。在 rultor.com 上，一个 DevOps 团队助手，我们使用 Docker 作为我们运行的每个构建的虚拟化技术。

    这是我们在容器启动后立即更改内部用户的方法。

    首先，这是我们如何启动一个新的Docker容器：

    请将以下段落从英文翻译成中文，不要更改专有名词：

    bash $ sudo docker run -i -t –rm -v “$(pwd):/main” \ yegor256/rultor /main/entry.sh

    当前目录中有两个文件：entry.sh和script.sh。entry.sh是Docker在启动时执行的文件，并且它包含以下内容：

    请将以下段落从英文翻译成中文，不要更改专有名词：

    adduser --disabled-password --gecos '' r
adduser r sudo
echo '%sudo ALL=(ALL) NOPASSWD:ALL' >> /etc/sudoers
su -m r -c /home/r/script.sh

    添加用户 --disabled-password --gecos '' r
添加用户 r sudo
将 '%sudo ALL=(ALL) NOPASSWD:ALL' 追加到 /etc/sudoers 文件中
su -m r -c /home/r/script.sh

    script.sh将作为容器中的用户r来执行。而这个r用户将拥有sudo权限。这正是所有使用rultor.com管理其DevOps流程的项目所需要的。
