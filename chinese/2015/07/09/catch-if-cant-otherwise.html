    Catch Me If You ... Can't Do Otherwise

    The following text is a translation of the original English article, performed by ChatGPT (gpt-3.5-turbo) and this Jekyll plugin:

    我不知道这是一个反模式还是一个常见且非常流行的错误，但我无处不见，必须写一写。我指的是没有重新抛出异常的异常捕获。我指的是类似这样的Java代码：

    try {
  stream.write(data);
} catch (IOException ex) {
  ex.printStackTrace();
}

    Pay attention: I don’t have anything against this code:

    try {
  stream.write('X');
} catch (IOException ex) {
  throw new IllegalStateException(ex);
}

    This is called exception chaining and is a perfectly valid construct.

    那么捕获异常并记录它有什么问题呢？让我们先来看看更大的画面。我们正在谈论面向对象的编程——这意味着我们在处理对象。下面是一个对象（确切地说，是它的类）的样子：

    最终类Wire { private final OutputStream stream; Wire(final OutputStream stm) { this.stream = stm; } public void send(final int data) { try { this.stream.write(x); } catch (IOException ex) { ex.printStackTrace(); } } }

    Here is how I’m using this class:

    new Wire(stream).send(1);

    看起来不错，是吧？当我调用send(1)时，我不需要担心那个IOException。它将在内部进行处理，如果出现问题，堆栈跟踪将被记录。但这种思维方式是完全错误的，它是从没有异常处理机制的语言（如C语言）继承而来的。

    例外是为了简化我们的设计而发明的，通过将整个错误处理代码从主逻辑中移出。此外，我们不仅仅是将其移出，还将其集中在一个地方——整个应用程序的入口点main()方法中。

    异常的主要目的是尽可能收集有关错误的信息，并将其上浮至最高级别，以便用户能够采取相应措施。异常链接能够进一步帮助我们在信息上进行扩展。每次捕获并重新抛出异常时，我们实际上是将我们的气泡（即异常）放入一个更大的气泡中。当它到达表面时，会有许多气泡，每个气泡都保留在另一个气泡内，就像一个俄罗斯套娃。原始异常是最小的气泡。

    当你捕捉到异常却不重新抛出时，你基本上是弹掉了泡泡。里面的一切，包括原始异常和所有包含信息的其他泡泡，都在你的手中。你不让我看到它们。你以某种方式使用它们，但我不知道是如何做的。你在幕后做些什么，隐藏了可能重要的信息。

    如果你对我隐瞒这个，我无法保证向我的用户诚实，并在问题发生时公开报告。我简单地无法再信任你的send()方法，而我的用户也不会信任我。

    By catching exceptions without re-throwing them, you’re basically breaking the chain of trust between objects.

    My suggestion is to catch exceptions as seldom as possible, and every time you catch them, re-throw.

    很不幸的是，Java的设计在许多地方都违背了这个原则。例如，Java有已检查和未检查异常，而在我看来应该只有已检查异常（必须捕获或声明为可抛出的异常）。此外，Java允许在单个方法中声明多个异常类型为可抛出异常，这也是一个错误；应该只声明一个类型。另外，在继承层次结构的顶部有一个泛型的Exception类，这在我的观点中也是错误的。除此之外，一些内置类不允许抛出任何已检查异常，比如Runnable.run()。Java中的异常问题还有很多其他的问题。

    But try to keep this principle in mind and your code will be cleaner: catch only if you have no other choice.

    P.S. Here is how the class should look:

    最终类Wire { private final OutputStream stream; Wire(final OutputStream stm) { this.stream = stm; } public void send(final int data) throws IOException { this.stream.write(x); } }
